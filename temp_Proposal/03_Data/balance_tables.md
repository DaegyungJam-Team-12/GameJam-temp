## 전진 점수와 구간 단계

| 항목 | 규칙 |
|---|---|
| 작업 시작 | 전진 점수 0 · 구간 단계 1 |
| 점수 획득 | 파괴한 얼음의 티어 점수 합산 |
| 단계 상승 | 누적 전진 점수 20점마다 +1 |
| 단계 상한 | 영구 해금 최고 빙질과 현재 항로 빙질 상한 중 낮은 값 |
| 항로 반영 | 작업 종료 시 획득 전진 점수를 현재 항로 진행에 1:1 반영 |
| 항로 완료 | `routeProgress >= routeTarget` |
| 초과 점수 | 다음 항로로 넘기지 않음 · 운항 보수는 전액 지급 |

## 연쇄 규칙

## 운항 보수와 정비 자금

- 소비 가능한 재화는 정비 자금 하나다.
- 얼음 파괴 시 재질 기본값 × 운항 보수 배율 × 운항 효율 보정을 누적한다.
- 누적액은 `이번 운항 보수`로 표시하고 60초 종료 시 정비 자금에 확정 반영한다.
- 항로 진행 초과분과 관계없이 이번 작업에서 획득한 운항 보수는 전액 지급한다.
- 화물·연료·편지·전진 점수는 소비 재화가 아니다.
- 기준 얼음의 보수 절대값은 기존 T1 기준 1을 유지하고 UI 화폐 기호는 UI 단계에서 확정한다.

## 선박 정비 비용 총람

P0 구성: 공통 루트 1개 + 공통 진행 3개 + 정비 계통 9개 = **구매 항목 13개, 공통 루트 제외 노드 12개.**

| 브랜치 | 비용 |
|---|---|
| 기본 파쇄 장비 | 10 |
| 공통 진행 | 운항 효율 50 · 300 · 1,800 / 청빙 대응 500 / 심빙 대응 8,000 |
| 직접 파쇄 | 25 · 150 · 900 / 80 · 600 / 8,000 |
| 보조 파쇄 | 100 / 600 / 2,500 |
| 연쇄 파괴 | 200 / 800 / 12,000 |

시연 데이터 프로필의 초기 비용은 `max(1, ceil(표준 비용 × 0.1))`로 만든다. 시연용 값을 코드 조건문으로 만들지 않고 별도 데이터 파일로 관리하며, D5 이후에는 10분 리허설 결과에 따라 독립적으로 튜닝한다.

## 구현 중 밸런스 검증

아래 수치는 구현 없이 확정하지 않는다.

- 항로별 실제 완료 시간과 전체 30~60분 목표
- 각 정비 항목의 구매율·구매 시점·미구매 종료 항목
- T1/T2/T3별 초당 전진 점수와 초당 운항 보수
- 직접·보조·연쇄 계통을 섞은 대표 구매 순서 3종의 성과
- 30초 대기 중 정비 완료율과 다음 작업 이탈률

특히 현재 전진 점수 1/2/3은 HP 배율 1/6/36보다 완만하므로 상위 빙질에서 항로 진행이 지나치게 느려지는지 확인한다. 문제가 확인되면 전진 점수·항로 목표·HP 중 하나 이상을 D10까지 함께 조정한다. 초기값 대비 ±30% 제한은 두지 않는다.
