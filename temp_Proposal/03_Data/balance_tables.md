## 항해

| 상황 | 시간 |
|---|---:|
| 최초 실행 | 즉시 (대기 없음) |
| 표준 | 30초 |
| 시연 | 10초 |

- 30초 항해 타이머는 정산 화면이 닫힌 뒤 시작한다.
- 항해 중에는 자동 파쇄·자동 재화 획득·자동 목적지 진행이 없다.
- 앱 종료 중에도 다음 쇄빙 가능 시각은 절대 시각으로 진행된다. 자금·목적지 진행은 변하지 않는다.

## 특수빙

| 종류 | 확률 | 내구도 | 자금 | 효과 요약 |
|---|---:|---:|---:|---|
| 결정빙 | 2.5% · T2 이상 | 기본과 동일 | 해당 단계 ×4 | 파편 5개가 더 낮은 단계 얼음을 즉시 파괴. 대상 없으면 소멸 |
| 균열빙 | 2% · 전 단계 | 기본 ×0.6 | 해당 단계 ×1 | 반경 120px에 현재 클릭 피해 ×3 · 최대 3단 연쇄 |

- 특수빙은 한 화면에 최대 2개만 존재한다.
- 특수 효과로 파괴된 얼음도 각각 정비 자금과 목적지 진행을 얻는다.

## 정비 자금 계산식

```text
정비 자금 증가 = 내림(
  단계 기본 지급액
  × 특수빙 배수
  × (1 + 정비 효율 단계 × 0.10)
)
```

- 정비 효율은 같은 종류끼리 더한다.
- 결정빙 배수와 정비 효율은 서로 곱한다.
- 현재 자금은 정수로 저장하며 0보다 작아질 수 없다.

## 선박 정비 비용 총람

P0 구성: 공통 장비 4개 + 직접 파쇄 4개 + 보조 파쇄 3개 + 연쇄 파괴 3개 = **14개 노드.** D04 구매값은 저장하고 다음 쇄빙 시작의 커서 원형 판정·점선 원에 함께 적용한다.

| 계통 | 비용 |
|---|---|
| 공통 장비 | C01 강화 장비 100 / C02 정비 효율 400·2,400·14,000 / C03 청빙 대응 1,200 / C04 심빙 대응 18,000 |
| 직접 파쇄 | D01 주 파쇄기 출력 300·900·2,700 / D02 고속 구동 600·1,800·5,400 / D03 과잉 파쇄 8,000 / D04 범위 확장 1,000·3,000·9,000 |
| 보조 파쇄 | S01 보조 파쇄기 500 / S02 다중 타격 3,000·9,000 / S03 표적 분석 15,000 |
| 연쇄 파괴 | H01 파편 비산 700·2,100·6,300 / H02 특수빙 증폭 4,000·12,000 / H03 빙판 붕괴 20,000 |

효과와 선행 조건은 `02_System/upgrade_tree.md`를 따른다. 한 번의 특수빙으로 최종 정비 여러 개를 건너뛰지 않도록 최종 비용을 15,000~20,000 구간에 둔다.

## 실제 조정 절차

1. 월요일(7/20) 기본 반복부터 작업별 파괴 수와 단계별 비율을 기록한다.
2. 첫 구매가 첫 작업 뒤 가능한지 확인한다.
3. 초반 1~2회 작업마다 구매가 생기는지 확인한다.
4. T2와 T3 해금 직후 수입 효율이 체감되는지 확인한다.
5. 목표 플레이타임에서 세 목적지에 도달하는지 확인한다.
6. 화요일(7/21) 통합본에서 1차 조정하고, 목요일(7/23) 전체 플레이를 기준으로 최종 수치를 확정한다.

## 자금 정산 지표

- 획득 정비 자금 (`earnedFunds`)
- 파괴한 얼음 수 (`destroyedCount`)
- 목적지 진행 증가량 (`destinationProgressGain`) — 목표 초과분 제외
- 목적지 도달 여부와 ID (`reachedDestination`, `destinationId`)

세 누적값(`stageEarnedFunds`, `stageDestroyedCount`, `stageDestinationProgressGain`)은 정산 화면에서 읽기만 하고 다음 쇄빙 작업 시작 시 0으로 초기화한다.
