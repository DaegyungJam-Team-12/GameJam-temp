## 덱의 한 문장

> **ICEBREAKER는 화면 맨 아래 게임 바에서 얼음을 깨고, 파괴 즉시 얻는 정비 자금으로 배를 강화해 세 목적지까지 가는 게임이다.**

## 2. 게임 개요 — 구조 요약

| 항목 | 내용 |
|---|---|
| 역할 | 마지막 쇄빙 보급선 ICEBREAKER의 선장 |
| 핵심 행동 | 최대 20개 얼음 클릭 또는 누르고 있기 |
| 플레이 | 60초 고정 · 60초 동안 파괴량 기록 |
| 장기 목표 | 섬마을 → 등대항 → 북쪽 기지 |
| 성장 | T1~T3 얼음 단계 · 선박 정비 14개 |
| 판 사이 | 30초 항해 · 정비·운항 현황 |
| 자금 | 얼음 파괴 즉시 · 정비 자금 1종 |
| 플레이타임 | 목표 30~60분 |

결론: **축소 게임 바에서 정비·운항 현황·쇄빙 시작을 바로 선택하고, 60초 동안 얼음을 부숴 세 목적지까지 간다.**

## 3. 화면 구성 — 축소 게임 바가 메인 진입점

1. 평소 화면 — 정비·운항 현황·쇄빙 시작을 고르는 게임 바 (800×72)
2. 정비·운항 현황 — 해당 화면으로 바로 확장, 중간 메뉴 없음
3. 쇄빙 시작 — 960×540 전체 화면, 상단 왼쪽 자금 · 중앙 타이머 · 오른쪽 설정

결론: **축소 바는 메인 메뉴다. 960×540은 쇄빙·정비·운항 현황·정산에 공통으로 쓰며, 쇄빙 중에는 정비·운항 현황 같은 관리 기능을 숨긴다.**

## 4. 플레이 흐름

1. **항해 · 30초** — 정비·운항 현황 확인, 30초 뒤 쇄빙 시작 활성, 자동 시작 없음
2. **쇄빙 작업 · 60초** — 얼음을 직접 파괴, 정비 자금 즉시 획득, 얼음마다 목적지 진행 +1
3. **작업 종료 · 1.2초** — 입력·추가 피해 차단, 배경 얼음판 파괴, 정산 화면 전환
4. **쇄빙 작업 정산** — 자금·파괴 수·진행 증가 확인, 1.2초 후 넘기기, 4초 후 자동 항해

정산 뒤 다시 30초 항해로 순환한다.

## 5. 파쇄 규칙 — 한 얼음판이 60초 스테이지

1. 얼음판 위 20개로 시작 — 첫 작업은 T1, 자유 배치
2. 부순 얼음은 다시 생성 — 0.12초 파괴 → 공백 → 0.18초 생성, 연쇄는 0.03초 간격 순차
3. 파괴 즉시 자금·목적지 진행 +1
4. 작업 종료·자금 정산 — 1.2초 파괴 연출 → 정산 화면

결론: **자금과 목적지 진행은 파괴 순간 반영되고, 작업 종료 뒤 정산에서 이번 작업 합계를 확인한다.**

## 6. 목적지 진행

1. 출발항 — 식료품·우편 적재, 첫 작업 즉시
2. 섬마을 — 식료품·우편 · 목표 120
3. 등대항 — 발전기 연료·의약품 · 목표 600
4. 북쪽 기지 — 기계 부품·우편 · 목표 2,400 · **도착 시 P0 완료**

결론: **얼음마다 목적지 진행 +1. 쇄빙 중에는 숨기고 정산에서 증가량을 확인한다. 축소 바에는 현재 목적지와 진행도를, 운항 현황에는 화물과 이후 목적지를 표시한다.**

## 7. 선박 정비 — 14개 고정형 연결 트리

- 사용 규칙: ① 항해 중 정비 버튼으로 바로 연다 ② 정비 자금으로 14개 영구 강화 구매 ③ 한 화면에서 선택·구매, 스크롤 없음
- 계통: 공통 / 직접 파쇄 / 보조 파쇄 / 연쇄 파괴 — 한 보드 안의 가지 이름
- 상태 범례: 보유 / 구매 가능 / 잠김, 선택 노드는 이중 외곽선
- 하단 상세 패널: 이름 · 현재/다음 효과 · 비용 · 구매 버튼 · 선행 조건

## 10. 타격·오디오

### 타격 연출 5종

1. 균열 단계 — 타격점에서 3단계로 성장
2. 파편·섬광 — 피해와 파괴 규모를 즉시 표시
3. 보상 숫자 — 첫 파괴 `정비 자금 +10` · 일반 `+80` · 3개 이상 합산
4. 미세 화면 진동 — 쇄빙 작업 화면, 설정에서 끌 수 있음
5. 얼음 파열음 — T1은 높게, T3는 낮고 무겁게

### 오디오 기준

얼음=짧고 선명 · 보상음=큰 연쇄만 · 엔진=낮고 안정 · **최초 실행 음소거**

### 60초 오디오 흐름

카운트다운 + 낮은 엔진음 → 타격·연쇄 중심 (큰 연쇄 때 환경음 낮춤) → 종료 직전 0.1초 정지 → 작업 종료음 → 정산 확인음 → 항구 도착 화음

## 14. 개발 참고 — 게임 상태 흐름

1. 항해 중 — 축소 런처 표시 · 시작 버튼 카운트다운
2. 쇄빙 준비 완료 — 자동 확장 없음 · 쇄빙 시작 버튼 활성
3. 화면 선택 — 정비·운항 현황 또는 쇄빙 시작
4. 카운트다운 — 3·2·1 · 입력 불가
5. 쇄빙 작업 — 정확히 60초 · 관리 UI 없음
6. 작업 종료 — 1.2초 · 입력/피해 차단 · 얼음판 파괴
7. 자금 정산 — 최소 1.2초 · 입력 시 진행 · 4초 자동 항해
8. 축소 화면 복귀 — 정산 종료 뒤 30초 항해 시작
