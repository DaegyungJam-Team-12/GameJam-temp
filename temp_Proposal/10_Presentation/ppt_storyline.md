# ICEBREAKER — PPT 전체 스토리라인과 슬라이드별 문안

> **상태: LOCKED · v1.0 · 2026-07-18**  
> 대상: 게임잼 기획자·개발자 3명·아티스트 1명  
> 목적: 팀 전원이 같은 게임을 이해하고 D1 작업을 바로 시작하게 한다.

## 덱의 한 문장

> **ICEBREAKER는 마지막 쇄빙 보급선으로 60초 동안 유빙을 부수고, 파쇄 성과만큼 항로를 전진해 세 항구를 다시 잇는 게임이다.**

## 전체 서사

```text
왜 얼음을 깨는가
→ 유저는 60초 동안 무엇을 하는가
→ 그 결과가 어떻게 항로와 성장으로 이어지는가
→ 화면·아트·오디오는 무엇을 전달하는가
→ 게임잼에서 어디까지 만들 것인가
→ 누가 어떤 파일을 언제 완성하는가
```

## 1. 표지 — 게임의 약속

### 제목

**ICEBREAKER**

### 부제

**얼어붙은 바다를 뚫고, 끊어진 항로를 잇는다.**

### 보조 정보

`정비 자금 12.4K · 60s · 항로 42%`

### 발표 핵심

이 게임은 단순히 얼음을 깨는 게임이 아니라, 얼음을 깨서 멈춘 항로를 다시 잇는 게임이다.

## 2. 게임 정의 — 한 장으로 이해시키기

### 제목

**화물을 실은 배가 얼음을 깨고 세 항구를 잇는다**

### 화면 문안

- 역할: 마지막 쇄빙 보급선 ICEBREAKER의 선장
- 핵심 행동: 최대 20개 유빙 클릭·홀드 파쇄
- 플레이: 60초 고정, 실패 없는 성과 측정
- 장기 목표: 섬마을 → 등대항 → 북쪽 기지
- 성장: T1~T3 빙질 대응 + 선박 정비 3개 계통
- 판 사이: 표준 30초 항해, 정비·항로 확인
- 재화: 운항 보수 → 정비 자금 1종
- 목표 완주: 신규 저장 기준 30~60분, D5~D10 튜닝

### 하단 결론

**설명문 없이 화물·항로·항구 불빛으로 목적을 보여준다. 같은 60초 안에서 파괴 규모가 커지는 것이 성장의 증거다.**

## 3. 세계관 전달 — 설명하지 않고 보여주기

### 제목

**말하지 않아도 목적이 보이는 화면**

### 왼쪽

**항해 중 — 화물을 싣고 항구로**

- 목적지
- 항로 진행
- 다음 유빙까지 시간
- 정비 자금

### 오른쪽

**쇄빙 작업 — 얼음이 항로를 막는다**

- 최대 20개 유빙
- 이번 운항 보수
- 전진 점수
- 현재 빙질과 최대 연쇄

### 하단 결론

**이야기는 대사나 프롤로그가 아니라, 배에 실린 화물·가까워지는 항구·지도 위 선박 아이콘으로 전달한다.**

## 4. 플레이 리듬 — 능동과 대기의 경계

### 제목

**판 중에는 능동, 판 사이에는 조용한 항해**

### 능동 구간

`클릭·홀드 → 특수빙 판단 → 연쇄 확장`

### 대기 구간

`정산 확인 → 선박 정비 → 다음 유빙 대기`

### 잠금 문장

**자동 공격·방치 수익·자동 항로 진행은 없다. 보조 파쇄는 클릭·파괴·연쇄가 있을 때만 발동한다.**

---

## 5. 코어 루프 — 한 번의 파쇄가 항구까지 이어진다

### 제목

**한 번의 파쇄가 항구까지 이어지는 순환**

### 순환

1. 유빙 도착 — 유저가 직접 시작
2. 쇄빙 작업 — 정확히 60초
3. 운항 정산 — 보수와 항로 진행 확정
4. 항해·정비 — 표준 초기값 30초
5. 다음 유빙 도착

### 양쪽 대비

- 능동 국면: 파쇄 손맛과 큰 연쇄가 성장의 증거
- 대기 국면: 작업 복귀, 게임은 조용히 항해만 진행

---

## 6. 60초 플레이 — 실패가 아닌 성장 측정

### 제목

**60초는 실패 제한시간이 아니라 성장 측정자다**

### 네 단계

1. 20개로 시작 — 자유 위치, 첫 작업 T1
2. 파괴 → 재스폰 — 파괴·공백·생성의 리듬
3. 더 단단한 빙질 — 전진 점수로 구간 단계 상승
4. 60초 정산 — 보수·점수 확정, 다음 항해 시작

### 성장 정의

**처음엔 얼음을 하나씩 부순다. 정비가 쌓이면 보조 타격과 연쇄가 이어지고, 같은 60초 안에서 화면 전체가 더 크게 무너진다.**

## 7. 얼음 콘텐츠 — P0에 필요한 변주

### 제목

**P0의 변주는 T1~T3와 특수빙 2종이면 충분하다**

### 기본 빙질

- T1 백빙: HP×1 · 보수×1
- T2 청빙: HP×6 · 보수×8
- T3 심빙: HP×36 · 보수×70
- T4·T5: P1 확장

### 특수빙

- 결정빙 2.5%: 파편 5개 추적, 하위 티어 즉시 파괴
- 균열빙 2%: 반경 피해, 최대 3단 연쇄
- 18회 미출현 시 확정, 화면 동시 최대 2개
- 재스폰 직후 0.25초 연쇄 면역

## 8. 항로 진행 — 세 목적지로 완주 구조 만들기

### 제목

**파쇄 결과가 곧 항로의 거리다**

### 진행

1. 출발항 — 식료품·우편 적재
2. 섬마을 — 식료품·우편, 목표 120
3. 등대항 — 발전기 연료·의약품, 목표 600
4. 북쪽 항로 — 기계 부품·우편, 목표 2,400
5. 북쪽 기지 — 목표값 도달 후 공통 도착, 전용 엔딩은 선택

### 잠금 문장

**전진 점수는 항로에 1:1 반영한다. P0는 고정 순서 3개이며 화물 선택·인벤토리·효과는 만들지 않는다.**

## 9. 강화 — 실제 배를 정비하는 화면

### 제목

**강화 트리는 배를 직접 만지는 정비 화면이다**

### 경제 흐름

`얼음 파쇄 → 운항 보수 → 정비 자금 → 같은 60초에서 더 빠르고 넓은 파괴`

### 장비 영역

- 공통 장비
- 직접 파쇄
- 보조 파쇄
- 연쇄 파괴

### 제작 규칙

**구매 항목은 13개지만 장비 영역은 4곳만 만든다. 노드마다 새 아이콘을 만들지 않고 텍스트와 레벨 점을 재사용한다.**

## 10. 상태 흐름 — 7단계와 마지막 5초 이벤트

### 제목

**한 번의 실행은 7개 상태와 하나의 경고 이벤트로 흐른다**

1. 항해 중
2. 유빙 구간 도착
3. 쇄빙 준비
4. 카운트다운
5. 쇄빙 작업
6. 운항 정산
7. 항로 갱신

마지막 5초는 별도 상태가 아니다. 입력과 파쇄를 유지한 채 타이머·성에·낮은 펄스만 강조한다.

### 프로필

- 최초 실행: 즉시
- 표준 초기값: 30초
- 시연 초기값: 10초
- 장기 대기: P0 플레이테스트 뒤 재검토

## 12. 아트 방향 — 레퍼런스의 역할 분리

### 제목

**차가운 바다와 따뜻한 불빛을 한 화풍으로 묶는다**

### 레퍼런스

- DREDGE: 어두운 바다에서도 읽히는 배·등대·불빛의 실루엣
- Spiritfarer: 부드러운 핸드페인팅과 따뜻한 사람의 흔적
- FAR: Lone Sails: 선박과 장비가 캐릭터처럼 보이는 화면

### 아트 잠금

**캐주얼 핸드페인팅 게임 화면 × 미형 선장 반신 1장. 얼음 실루엣 6종은 재질을 바꿔 재사용하고, 항구 3종은 거리와 안개로 변주한다.**

음성 대사는 없고 최초 실행은 음소거다. 일정 부족 시 BGM부터 컷한다.

## 14. P0/P1 — 게임잼의 경계

### 제목

**P0는 고정 항로 3개를 완주하는 한 게임이다**

### P0

- 창·UI: 도킹/확장, 저장, KO/EN, Windows
- 파쇄: 60초, 최대 20개, T1~T3, 특수빙 2종
- 메타: 고정 항로 3개, 정비 자금, 구매 13개
- 전달: 화물, 항구 3곳, 공통 도착 연출
- 완료: 목표값 도달 → 북쪽 기지 공통 도착, 목표 30~60분
- 선택 엔딩: RC가 안정됐을 때만 스프린트 최후위로 추가

### P1

T4·T5, 흑빙·서리빙, 추가 목적지, 계약·분기 항로, 화물 보정, macOS, 다국어, Steam 제품화.

### 기본 제외

자동 공격, 앱 종료 중 수익, 거래 경제, 선체 HP, 장문 대화, 물류 시뮬레이션.

## 15. 밸런스 프로필 — 표준과 시연 분리

### 제목

**초기 수치는 구현과 플레이테스트로 검증한 뒤 잠근다**

| 항목 | 표준 | 시연 |
|---|---:|---:|
| 쇄빙 작업 | 60초 | 60초 |
| 다음 유빙 | 30초 | 10초 |
| 항로 목표 | 120 / 600 / 2,400 | 40 / 120 / 300 |
| 정비 비용 | 초기값 100% | `ceil(표준 × 0.1)` 초기값 |
| 목표 | 30~60분 목표 | 10분 이내 리허설 |

### 잠금

같은 코드와 저장 구조를 사용하며 데이터 JSON만 교체한다. 저장 파일은 표준/시연으로 분리하고 공개 설정에는 프로필 전환을 노출하지 않는다. D5~D10에는 제한 없이 튜닝하고 D11에 잠근다.
