# ICEBREAKER — 콘셉트 잠금

> **상태: LOCKED · 2026-07-19 · 기준 문서: `new/ICEBREAKER_SPEC_v2.md` v2.6**  
> 이후 문서와 PPT에서 세계관·플레이 의미·용어가 충돌할 경우 통합 기획서를 우선한다. 수치와 구현 세부는 `02_System/`, `03_Data/`를 따른다.

## 1. 게임 정의

### 한 줄 소개

**작업 표시줄 위의 쇄빙선을 열어 60초 동안 얼음을 부수고, 정비 자금으로 배를 강화해 세 목적지까지 가는 PC 게임.**

### 유저 대상 카피

**얼어붙은 바다를 뚫고, 끊어진 항로를 잇는다.**

### 장르

- 60초 능동 파쇄 액션
- 영구 강화형 증분 게임
- taskbar 상주형 데스크톱 컴패니언
- 상시 자동 전투 없는 직접 조작 중심 게임

### 핵심 플레이 판타지

1. **파쇄:** 여러 얼음을 빠르고 시원하게 부순다.
2. **성장:** 선박 정비 13개로 같은 60초 동안 더 넓고 단단한 얼음을 처리한다.
3. **항해:** 얼음을 부순 만큼 목적지 진행이 쌓이고, 결국 화물을 기다리는 항구에 도착한다.

## 2. 팀이 알고 있는 세계관

수년간 이어진 한파로 북쪽 군도의 바다가 얼어붙었다. 대형 화물선과 여객선의 운항은 중단되었고, 섬들은 연료·의약품·기계 부품·우편을 안정적으로 공급받지 못한다.

소형 쇄빙선 **ICEBREAKER**는 이 해역에서 정기적으로 운항할 수 있는 마지막 보급선이다. 플레이어는 ICEBREAKER의 젊은 선장이 되어 얼음으로 막힌 바닷길을 열고 섬과 섬을 다시 연결한다.

선장은 섬에서 자랐고, 어린 시절 자신도 보급선이 들어오는 날을 기다렸다. 지금은 직접 배를 몰아 다른 섬에 필요한 물자를 전달한다. 이 배경은 캐릭터의 태도와 키비주얼을 위한 내부 설정이며, 게임 시작 시 텍스트로 설명하지 않는다.

세계는 춥고 위험하지만 정서는 절망이나 멸망이 아니다. 핵심 대비는 다음과 같다.

> **차가운 바다 위에서 사람과 사람을 다시 연결하는 작은 배**

### 의도적으로 정의하지 않는 것

- 한파가 시작된 정확한 원인
- 국가·정부·전쟁·정치 상황
- 인류 전체의 생존 여부
- 선장의 상세한 과거사와 가족 관계
- 항구별 장문의 사연

게임이 직접 다루지 않는 설정은 만들지 않는다.

## 3. 유저에게 세계관을 전달하는 방식

세계관은 프롤로그·대화·컷신으로 설명하지 않는다. 화면에서 다음 요소만 반복해 보여 준다.

- 작업 표시줄 위에 머무는 작은 선박 게임 바
- 멀리 보이는 다음 목적지의 불빛
- 배에 적재된 화물 이름
- 얼음이 부서질수록 앞으로 나아가는 배
- 축소 게임 바에서 바로 여는 선박 정비와 운항 현황 화면

### 전달 규칙

- 플레이 중 세계관 설명문을 띄우지 않는다.
- 한 화면의 상황 문구는 원칙적으로 한 줄을 넘기지 않는다.
- NPC 감사 대화와 스테이지별 스토리 삽화를 만들지 않는다.
- 목적지 도착 연출은 항구 이름 표식과 불빛 변화의 1~2초 공통 시스템 연출로 만든다.
- 캐릭터 원화는 타이틀·정비 화면에서 사용하고, 반복 대사창으로 사용하지 않는다.
- 화물은 P0에서 목적을 전달하는 시각 정보이며, 별도의 인벤토리·적재·소비 시스템으로 만들지 않는다.

## 4. 60초 플레이의 세계관상 의미

- **쇄빙 작업:** 막힌 항로의 얼음을 제거하는 60초 능동 플레이.
- **항해:** 작업을 마친 배가 다음 얼음 구간까지 이동하는 30초 대기 구간.

한 번의 60초는 항해 전체가 아니라 **다음 수로를 확보하기 위한 한 차례의 쇄빙 작업**이다.

### 타임업은 실패가 아니다

- 60초는 선박 파괴나 연료 고갈까지 남은 시간이 아니다.
- 시간이 끝나면 해당 쇄빙 작업을 마치고 자금 정산 화면에서 성과를 확인한다.
- 하트·선체 HP·게임 오버·시간 연장 기능을 추가하지 않는다.
- 목적지 목표를 도중에 달성해도 남은 시간 동안 정비 자금을 계속 획득한다.

### 목적지 진행

- 얼음 하나를 파괴할 때마다 현재 목적지 진행도가 `+1` 증가한다.
- 얼음 단계, 특수빙, 파괴 원인은 진행 증가량에 영향을 주지 않는다.
- 세 목적지 목표는 각각 120, 600, 2,400이다. 초과분은 다음 목적지로 넘기지 않는다.
- 목적지 도착 여부는 작업 종료 때 확인한다.

---

## 5. 판 사이 항해의 의미

쇄빙 작업을 마친 배는 확보한 수로를 따라 다음 구간까지 항해한다. 이 시간이 480×56 축소 게임 바 상태의 대기 구간이다.

- 표준 30초 · 시연 데이터 10초
- 최초 실행은 항해 대기 없이 바로 쇄빙 가능
- 항해 중 축소 게임 바에서 선박 정비·운항 현황 확인 가능
- 30초가 끝나도 자동으로 창을 펼치지 않고 쇄빙 시작 버튼만 활성화
- 유저가 직접 눌러야 다음 60초 쇄빙 작업 시작
- 항해 중에는 자동 파쇄와 자동 재화 획득이 없음

축소 게임 바는 정비 자금·목적지 진행·정비·운항 현황·쇄빙 시작·설정을 한곳에 제공하는 메인 진입점이다.

---

## 6. 얼음 단계와 해금

얼음 단계 해금은 광산의 더 깊은 층을 여는 것이 아니라, 선박이 더 단단한 얼음을 상대할 수 있도록 장비를 보강하는 과정이다.

| 단계 | 화면 이름 | 세계관상 의미 | 등장 조건 |
|---|---|---|---|
| T1 | 백빙 | 항구 인근의 얇은 유빙 | 처음부터 |
| T2 | 청빙 | 외해에서 오래 압축된 단단한 얼음 | 선박 정비 `청빙 대응` 구매 후 |
| T3 | 심빙 | 북쪽 빙산에서 갈라진 짙고 단단한 얼음 | 선박 정비 `심빙 대응` 구매 후 |

목적지 도달은 T2·T3를 해금하지 않는다. 새 얼음 단계는 선박 정비만 해금한다. T4·T5는 P1 확장 후보로 둔다.

---

## 7. 재화의 의미

소비 가능한 재화는 **정비 자금** 하나만 사용한다. 화물·연료·편지는 재화나 인벤토리가 아니다.

- 정비 자금은 얼음 속에서 발견되는 물건이 아니다. 막힌 항로의 얼음을 제거한 작업량에 따라 확보되는 선박 유지 비용이다.
- 얼음을 파괴한 **순간** 정비 자금이 즉시 지급된다. 동전이나 부품을 줍는 과정은 없다.
- 이 설정을 별도의 이야기 창으로 설명하지 않고 `얼음 파괴 → +금액 → 선박 정비`의 연결로 이해시킨다.
- 작업 종료 뒤 자금 정산 화면은 이미 지급된 이번 작업 자금과 성과를 요약하며 추가 지급은 하지 않는다.

---

## 8. 강화의 의미

강화는 추상적인 스킬이 아니라 ICEBREAKER의 파쇄 장비를 정비·개조하는 과정이다. 선박 정비 13개는 네 계통으로 나뉜다.

- **공통 장비:** 계통 개방, 정비 효율, 청빙·심빙 대응
- **직접 파쇄:** 한 목표를 빠르게 깨는 주 파쇄 장비
- **보조 파쇄:** 플레이어의 직접 입력에 반응해 추가 타격하는 보조 장비
- **연쇄 파괴:** 충격과 파편을 주변 얼음으로 전달하는 장비

보조 파쇄는 사용자가 입력하는 동안에만 충전되고 작동한다. 자동 공격과 방치 수익은 없다.

정비는 쇄빙 작업 밖에서만 구매한다. 30초 항해가 끝난 뒤에도 쇄빙 시작을 누르기 전까지 구매할 수 있다. 항구는 화물 전달을 보여주는 장소이며, 강화 루프를 막는 체크포인트가 아니다.

## 9. 단기 루프와 장기 루프

### 단기 루프 — 한 번의 플레이

```text
항해 및 정비 대기 (30초)
          ↓ 유저가 직접 쇄빙 시작
[3·2·1 카운트다운]
          ↓
[쇄빙 작업 60초]
 얼음 파괴 → 즉시 정비 자금 + 목적지 진행 +1
          ↓ 작업 종료 1.2초
[자금 정산 화면]
 획득 자금 · 파괴 수 · 목적지 진행 증가량 요약
          ↓ 최소 1.2초 · 최대 4초
[축소 화면 복귀 · 다음 항해 30초]
          └────────────── 반복
```

### 장기 루프 — 세 목적지

```text
화물과 목적지 확인
→ 쇄빙 작업 반복으로 목적지 진행 누적
→ 목표 도달 · 작업 종료 시 도착 확인
→ 항구 이름과 1~2초 도착 연출
→ 다음 목적지 진행 시작
```

P0에서 화물 선택과 자유 항로 선택은 구현하지 않는다. 정해진 목적지와 화물을 순서대로 보여주며, 선택형 운송 시스템은 P1 후보로 둔다.

---

## 10. P0 진행과 선택 엔딩

P0의 진행은 세 목적지를 순서대로 여는 하나의 선형 여정이다.

1. **섬마을** · 식료품·우편 · 목표 120
2. **등대항** · 발전기 연료·의약품 · 목표 600
3. **북쪽 기지** · 기계 부품·우편 · 목표 2,400

### 북쪽 기지 도착과 선택 엔딩

- 북쪽 기지 도착이 P0 완료 조건이다.
- 도착 시 다른 목적지와 같은 공통 도착 연출(항구 이름 표식·불빛 변화)을 사용한다.
- 별도 엔딩 장면은 제출 후보가 안정된 뒤에만 마지막 순서로 검토한다.

## 11. 화면과 아트가 전달해야 할 정서

### 시각 대비

- 바다·얼음·밤: 남색, 청록색, 푸른색
- 선박·항구·사람의 흔적: 주황색, 노란색, 따뜻한 흰색
- 게임 화면: 작은 화면에서 읽히는 캐주얼 손그림풍 (도트 그래픽 사용 안 함)
- 캐릭터·키비주얼: 미형의 원화 스타일

### 화면이 말해야 하는 것

1. 이 배는 물건을 싣고 있다.
2. 앞의 얼음 때문에 목적지에 가지 못하고 있다.
3. 얼음을 깨면 배가 항구에 가까워진다.

이 세 가지가 보이면 별도의 설정 설명은 추가하지 않는다.

## 12. 표준 용어

| 의미 | 사용 용어 |
|---|---|
| 60초 플레이 | 쇄빙 작업 |
| 개별 대상 제거 | 얼음 파괴 |
| 보유 재화 | 정비 자금 |
| 30초 대기 | 항해 |
| 60초 종료 상태 | 작업 종료 |
| 종료 후 성과 요약 | 자금 정산 |
| 영구 성장 | 선박 정비 |
| 입력 연동 보조 공격 | 보조 파쇄 |
| 메타 진행 | 목적지 진행 |
| 목적지 정보 화면 | 운항 현황 |

### 사용하지 않는 표현

- 채굴, 심도, 깊이 내려간다, 채굴층
- 골드, 운항 보수, 전진 점수, 구간 단계
- 재결빙
- 자동화, 방치 수익

## 13. 최종 잠금 문장

> **ICEBREAKER는 얼어붙은 바다에서 마지막 운항을 이어가는 쇄빙선을 조작해, 60초 동안 얼음을 부수고 파괴한 만큼 정비 자금과 목적지 진행을 즉시 얻으며, 고립된 섬에 연료와 의약품, 편지를 전달하는 파쇄 증분 게임이다. 이야기는 대사와 컷신이 아니라 화물, 목적지의 불빛과 배의 전진으로 전달한다.**
