# UI / UX — 항로와 화물로 이야기를 전달하는 화면

> **상태: LOCKED · v3.0 · 2026-07-18**  
> 기준: `01_Concept/concept.md`, `02_System/game_states.md`, `02_System/route_board.md`  
> 목표: 설명문 없이도 유저가 **화물을 실은 쇄빙선이 얼음을 부수며 항구로 가고 있다**고 이해하게 한다.

## 1. UX 원칙

### 게임플레이가 먼저 보인다

- 60초 쇄빙 화면을 대화창·미션 카드·결과 모달로 가리지 않는다.
- 플레이 영역은 바다·얼음·배가 하나의 장면으로 보이게 구성한다.
- UI 패널은 항해 계기처럼 조용하게 붙이고, 카드 그리드나 두꺼운 테두리를 반복하지 않는다.
- 플레이 중 반드시 필요한 수치만 고정 표시하고 나머지는 상태에 따라 교체한다.

### 이야기는 플레이 정보에 포함한다

- `목적지`는 왜 전진하는지를 알려준다.
- `화물`은 누가 이 배를 기다리는지를 암시한다.
- `항로 진행`은 반복 플레이가 어디로 이어지는지 보여준다.
- 멀리 있는 항구 불빛은 목표를 시각적으로 유지한다.
- 항구 도착과 화물 하역은 장기 진행의 보상이다.

### 설명 예산

- 플레이 중 세계관 설명문 없음
- 한 화면의 상황 문구는 원칙적으로 한 줄
- 튜토리얼은 조작 안내 3문장 이내
- 항구 도착 연출은 1~2초
- 캐릭터 말풍선·NPC 감사 대화·장문의 항해일지 없음

## 2. 화면 규격

### 도킹 상태

- 크기: **480×56px**
- 위치: Windows taskbar 바로 위
- 창 전체가 클릭 가능한 인터랙션 영역
- 알파 클릭 통과 사용 안 함
- 소리 기본 꺼짐
- 알림·팝업·자동 확장 없음

### 확장 상태

- 크기: **720×480px**
- 기준 해상도: 1920×1080
- 최소 지원: 1366×768
- taskbar 위에서 위쪽으로 전개

```text
┌──────────────────────────────────────────────────────────┐
│ 목적지 · 상태              00:42              이번 운항 │ 48
├──────────────────────────────────────────┬───────────────┤
│                                          │               │
│       유빙 구간 520×376                  │ 상태별 패널    │
│       바다 + 먼 항구 + 쇄빙선 + 얼음      │ 200×376        │
│                                          │               │
├──────────────────────────────────────────┴───────────────┤
│ 정비 자금 │ 선박 정비 │ 항로 │ 설정 │ 접기              │ 56
└──────────────────────────────────────────────────────────┘
```

### 공통 치수

- 상단 상태 바: 48px
- 쇄빙 화면: 520×376px
- 오른쪽 상태 패널: 200×376px
- 하단 내비게이션: 56px
- 내부 기본 여백: 12px
- 버튼 최소 클릭 영역: 44×44px
- 본문 최소 크기: 14px
- 주요 버튼·수치: 16px 이상
- 60초 타이머: 28px 이상

## 3. 정보 우선순위

### 항상 보여야 하는 정보

1. 현재 상태 또는 목적지
2. 60초 타이머 또는 다음 유빙 구간까지 시간
3. 이번 운항 보수 또는 보유 정비 자금
4. 항로 진행

### 쇄빙 작업 중에만 보여야 하는 정보

- 현재 빙질
- 전진 점수
- 현재 연쇄
- 활성 보조 장비

### 작업 사이에만 보여야 하는 정보

- 화물
- 선박 정비
- 항로 지도
- 이전 기록 비교

같은 정보를 상단 바와 오른쪽 패널에 중복 표시하지 않는다.

## 4. 화면 흐름

```text
[도킹 · 항해 중]
        ↓ 다음 유빙 구간 도착
[도킹 · 쇄빙 가능]
        ↓ 클릭
[쇄빙 준비]
        ↓ 쇄빙 시작
[카운트다운 3·2·1]
        ↓
[60초 쇄빙 작업]
        ↓ 시간 종료
[운항 정산 · 항로 갱신]
        ├─ 항로 진행 중 → [항해 중]
        └─ 항로 완료   → [1~2초 항구 도착] → [다음 항로 준비]
```

항구 도착은 별도 스토리 화면이 아니라 운항 정산의 조건부 전환이다.

## 5. 도킹 스트립 — 480×56

도킹 스트립은 작은 상태 표시줄이 아니라 **바다를 이동하는 배의 압축 화면**으로 보이게 한다.

### 항해 중

```text
┌────────────────────────────────────────────────────┐
│ 🚢  등대항 42% ━━━━━━━░░   다음 유빙 00:24   12.4K │
└────────────────────────────────────────────────────┘
```

- 왼쪽: ICEBREAKER 아이콘 또는 작은 선박 실루엣
- 중앙: 목적지와 항로 진행 바
- 오른쪽: 다음 유빙 구간까지 시간과 정비 자금
- 진행 바 위에서 작은 선박 아이콘이 오른쪽으로 이동
- 얼음 실루엣은 타이머가 끝날수록 오른쪽에서 천천히 나타남
- 애니메이션은 낮은 프레임으로 동작하고 업무 화면의 시선을 끌지 않음

### 유빙 구간 도착

```text
┌────────────────────────────────────────────────────┐
│ 🚢  유빙 구간 도착        쇄빙 가능 ›       12.4K │
└────────────────────────────────────────────────────┘
```

- 전체 스트립을 클릭하면 확장
- 팝업·OS 알림·사운드 재생 없음
- 선박 불빛과 얼음 윤곽이 한 번 천천히 밝아지는 정도로 준비 상태 표시
- 유저가 누르기 전까지 자동 시작하지 않음

### 항구 도착 직후

```text
┌────────────────────────────────────────────────────┐
│ 🚢  등대항 도착          다음 출항 준비 00:24      │
└────────────────────────────────────────────────────┘
```

항구에서 다음 항로로 넘어갈 때만 `항해 중` 대신 `다음 출항 준비`를 사용한다.

## 6. 쇄빙 준비 화면

첫 실행과 매 유빙 구간 도착 후 표시한다. 스토리 브리핑이 아니라 현재 운항 정보를 확인하는 화면이다.

### 상단

```text
목적지  등대항                 항로 진행 42%       정비 자금 12.4K
```

### 쇄빙 화면 영역

- 바다와 목적지 실루엣을 보여준다.
- 쇄빙선 뱃머리는 화면 아래 중앙에 낮게 배치한다.
- 다음 작업에서 등장할 얼음은 성에 낀 실루엣으로 미리 보인다.
- 배경은 플레이 화면과 동일하게 유지해 별도의 브리핑 화면처럼 보이지 않게 한다.

### 오른쪽 패널

```text
등대항

화물
연료   의약품

항로
42%  ━━━━━░░░░

[ 쇄빙 시작 ]
```

- 화물은 최대 3개 아이콘과 짧은 명칭만 표시
- 화물 무게·가격·내구도·적재량 없음
- P0에서는 화물을 선택하거나 교체할 수 없음
- `쇄빙 시작`을 유일한 강조 버튼으로 사용

### 하단

```text
정비 자금 12.4K    선박 정비    항로    설정    접기
```

## 7. 카운트다운

- 3초 동안 얼음의 성에 실루엣이 실제 스프라이트로 드러난다.
- 중앙 숫자는 `3 → 2 → 1`만 표시한다.
- `시작!`, 세계관 문장, 캐릭터 대사를 추가하지 않는다.
- 마지막 숫자와 동시에 얼음 입력과 타이머를 활성화한다.
- 하단 내비게이션과 오른쪽 패널 입력은 잠근다.

## 8. 60초 쇄빙 화면

### 장면 구성

- 바다 배경이 매우 느리게 아래로 흐르며 선박의 전진을 암시한다.
- 현재 목적지의 불빛이나 실루엣이 화면 상단 원경에 보인다.
- 쇄빙선 뱃머리는 화면 아래 중앙의 배경 레이어에 고정한다.
- 얼음은 배와 UI보다 위에 보이며 기존 자유 위치 스폰 규칙을 유지한다.
- 뱃머리 이미지는 `Raycast Target`을 끄고 클릭을 가로막지 않는다.
- 플레이 영역 안에 카드·설명 박스·캐릭터 초상화를 올리지 않는다.

### 상단 상태 바

```text
등대항 항로                         00:42          이번 운항 +1,240
```

- 중앙: 가장 큰 타이머
- 왼쪽: 목적지
- 오른쪽: 누적 운항 보수
- 마지막 5초 이벤트에서는 입력과 파쇄를 유지한 채 타이머만 커지고 색·성에·낮은 펄스로 긴장감을 전달

### 오른쪽 패널

```text
항로 진행
42%  ━━━━━░░░░

전진 점수     +32
현재 빙질      T2
현재 연쇄      ×7
최고 연쇄      ×12

보조 장비      충전 7/10
```

- 특수빙 설명은 처음 발견했을 때 패널 하단에 2초간 한 줄만 표시
- 반복 출현 시 아이콘과 효과 색만 보여줌
- 활성 보조 장비가 없으면 해당 행을 숨겨 빈 공간을 확보

### 하단 내비게이션

- 정비·항로·접기 버튼 비활성
- 버튼은 흐리게 표시하되 사라지지 않게 해 상태 변화를 예측 가능하게 함
- 음소거와 화면 진동 토글만 즉시 조작 가능
- 타이머는 포커스를 잃어도 계속 진행하며 일시정지하지 않음

---

## 9. 운항 정산과 항로 갱신

전체 화면 결과 모달을 사용하지 않는다. 플레이 장면은 얼어붙은 상태로 남기고 오른쪽 패널만 정산 정보로 전환한다.

### 기본 전환

1. 0초: 남은 얼음 이동 정지
2. 0.0~0.3초: 화면과 사운드 짧게 정지
3. 0.3~0.7초: 성에 프레임이 갈라져 떨어짐
4. 0.7~1.3초: 항로 진행 바와 지도 위 선박 아이콘이 앞으로 이동
5. 1.3초 이후: 정산 패널 조작 가능

### 오른쪽 정산 패널

```text
운항 정산

운항 보수       +1,240
파괴한 얼음         86
최고 빙질           T2
최대 연쇄          ×12
전진 점수          +96

항로 진행
42% → 58%

[ 선박 정비 ]
```

- 직전 쇄빙 작업보다 운항 보수가 늘었으면 `+18%`만 작은 보조 정보로 표시
- 칭찬 문구·등급·별점·보상 상자를 추가하지 않는다.
- 다음 유빙 구간까지 남은 시간을 패널 하단에 표시한다.
- 정산을 확인하는 동안에도 대기 타이머는 진행된다.

## 10. 항구 도착

항구 도착은 일반 정산 전환을 재사용하는 조건부 연출이다.

### 연출 순서 — 최대 2초

1. 성에 프레임이 갈라지고 원경의 항구가 근경으로 전환
2. 짧은 뱃고동과 함께 항구 불빛이 켜짐
3. 화물 아이콘 1~3개가 선박에서 부두 방향으로 이동
4. 항로 지도에서 목적지 노드가 주황색으로 점등
5. 화면 중앙에 `등대항 도착` 한 줄 표시

### 하지 않는 것

- NPC 등장과 감사 대사
- 화물별 전용 컷신
- 항구마다 다른 조작 방식
- 화면을 가득 채우는 보상 팝업
- 스토리 챕터 설명

P0에서는 같은 도착 연출을 모든 항구에 재사용하고 배경·항구 이름·화물 아이콘만 교체한다.

## 11. 항로 화면

하단 `항로` 버튼으로 작업 사이에만 열 수 있다. P0 항로는 고정 순서이며 선택 화면이 아니라 진행 확인 화면이다.

```text
출발항 ───── 섬마을 ───── 등대항 ───── 북쪽 기지
                           🚢

현재 목적지  등대항
화물         연료 · 의약품
진행         58%
```

- 완료한 항구 노드는 주황색으로 점등
- 현재 항로는 밝은 선, 아직 열리지 않은 항로는 흐린 선으로 표시
- 선박 아이콘은 `routeProgress / routeTarget` 비율로 선 위를 이동
- P0에서는 지도 드래그·확대·항로 선택 없음
- P1에서만 계약 선택과 분기 항로를 검토

항로 화면에 세계 지도·정치 정보·항구 사연을 추가하지 않는다.

## 12. 선박 정비 화면

선박 정비는 추상적인 인포그래픽 트리 대신 실제 배를 개조하는 화면처럼 구성한다.

### 기본 구성

```text
┌──────────────────────────────────────────┬───────────────┐
│                                          │ 선택 장비      │
│        ICEBREAKER 측면도                  │               │
│                                          │ 주 파쇄기 출력 │
│   [주 파쇄기] [보조 장비] [충격 장치]     │ LV.2 / 3      │
│                 [공통 장비]               │ 비용 900       │
│                                          │ [ 강화 ]       │
└──────────────────────────────────────────┴───────────────┘
```

- 중앙 배의 장비 위치를 눌러 브랜치를 선택
- 공통 진행 / 직접 파쇄 / 보조 파쇄 / 연쇄 파괴 4개 영역
- 선택한 영역의 3개 노드만 오른쪽 패널에 표시
- 모든 노드를 한 화면에 선으로 연결한 거대한 도표는 사용하지 않음
- 선행 조건은 노드 사이의 짧은 연결선과 잠금 아이콘으로만 표현
- 강화 구매 시 해당 선박 부위가 짧게 점등하거나 외형 레이어가 바뀜
- 캐릭터 원화는 화면 한쪽의 장식 요소로만 사용하고 장비 설명 말풍선을 붙이지 않음

### 상태 규칙

- 항해 중·유빙 구간 도착·쇄빙 준비·항로 갱신 상태에서 구매 가능
- 카운트다운과 60초 쇄빙 작업 중 구매 불가
- 구매 불가 상태에서는 비용 버튼을 비활성화하고 이유를 한 줄로 표시
- 정비 자금 잔액과 구매 후 예상 잔액을 함께 표시

## 13. 목적지 거리의 환경 표현

목적지마다 여러 장의 전용 배경을 그리지 않는다. 하나의 항구 실루엣을 거리와 안개 값으로 조절한다.

| 항로 진행 | 배경 상태 |
|---:|---|
| 0~32% | 작은 불빛만 보임 · 안개 강함 |
| 33~65% | 등대·건물 실루엣 확인 가능 |
| 66~99% | 방파제와 항구 형태가 보임 |
| 100% | 항구 근경 · 도착 상태 |

배경 전환은 쇄빙 작업 중 실시간으로 일어나지 않는다. 운항 정산 후 다음 작업을 열 때 단계가 바뀐다. 플레이 중 갑자기 배경이 튀는 것을 막는다.

## 15. 첫 실행 튜토리얼

세계관 프롤로그 없이 첫 유빙 구간 도착 상태에서 시작한다.

1. 첫 얼음 위: `얼음을 눌러 파쇄`
2. 첫 파괴 후: `누르고 있으면 연속 발사`
3. 첫 운항 정산: `정비 자금으로 배를 강화`

- 각 문장은 해당 행동을 하면 즉시 사라진다.
- 다시 보기 옵션 외에는 반복하지 않는다.
- 항로·화물·세계관은 튜토리얼 문장으로 설명하지 않는다.

## 17. 접근성과 업무 불간섭

- 소리 기본 꺼짐
- 포커스 스틸 금지
- OS 알림과 팝업 금지
- 도킹 스트립 자동 확장 금지
- 준비 완료 상태에서 반복 점멸 금지
- 화면 진동 강도 조절과 완전 끄기 제공
- 빙질은 색뿐 아니라 실루엣·내부 무늬·명칭으로 구분
- 특수빙은 색 외에 고유 아이콘과 표면 패턴 사용
- 클릭 가능한 얼음은 호버 시 외곽선 또는 밝기 변화
- 중요한 정보는 호버에만 의존하지 않음
- KO/EN 문자열은 짧게 유지하고 한 줄 제목이 두 줄로 줄바꿈되지 않게 함
- 쇄빙 작업 중 포커스를 잃어도 타이머는 계속 진행
- 작업 중 접기·정비·항로 전환 비활성

## 18. UI가 요구하는 데이터

화물 시스템을 만들지 않고 다음 표시 데이터만 둔다.

| 데이터 | 용도 |
|---|---|
| `routeId` | 현재 항로 식별 |
| `routeName` | 항로 화면 표시 |
| `destinationName` | 준비·플레이·도착 화면 표시 |
| `cargoIconIds[3]` | 최대 3개 화물 아이콘 |
| `cargoLabels[3]` | 화물 짧은 명칭 |
| `routeProgress` | 현재 누적 전진 점수 |
| `routeTarget` | 항구 도착 목표값 |
| `routeProgressRatio` | 항로 바·지도 위치·배경 거리 단계 |
| `nextEncounterReadyAt` | 도킹 타이머 |
| `arrivalFlag` | 항구 도착 연출 분기 |
| `finalRouteFlag` | 최종 항로 여부 |
| `t3FirstBreakFlag` | 최고 빙질·성장 이정표 기록 |

북쪽 기지도 `routeProgress >= routeTarget`일 때 공통 항구 도착 연출로 완료한다. 전용 엔딩 화면은 P0 필수 UI가 아니며 RC가 안정됐을 때만 스프린트 최후위 옵션으로 추가한다.

화물은 아이콘과 이름만 표시한다. 수량·무게·효과 데이터는 두지 않는다.

## 19. P0와 P1 경계

### P0

- 고정 순서 항로 지도
- 목적지·화물 표시
- 항로 진행 바와 선박 아이콘
- 거리 3단계 + 도착 1단계 배경 표현
- 재사용 가능한 항구 도착 연출 1종
- 상태에 따라 내용이 바뀌는 오른쪽 패널 1개
- 선박 부위 선택형 정비 화면
- 조작 튜토리얼 3문장

### P1

- 자유 항로 선택
- 운송 계약 선택
- 화물별 게임플레이 효과
- 항해일지와 항구 기록
- 항로 분기
- 상세 통계 그래프

P1 항목을 암시하기 위한 비활성 버튼이나 빈 슬롯을 P0 화면에 미리 만들지 않는다.

## 20. 완료 기준

처음 보는 유저가 대사나 설명문 없이 다음 네 가지를 이해하면 성공이다.

1. 나는 쇄빙선을 조작한다.
2. 앞의 얼음을 깨야 항구로 갈 수 있다.
3. 배에는 전달할 화물이 있다.
4. 정비하면 더 단단한 얼음과 더 먼 항로에 도전할 수 있다.

플레이테스트에서는 첫 화면을 5초 보여준 뒤 이 네 질문에 답하게 한다. 세 개 이상 맞히면 콘셉트 전달 성공으로 판정한다.
