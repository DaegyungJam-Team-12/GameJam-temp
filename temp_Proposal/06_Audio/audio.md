# ICEBREAKER — 오디오 디렉션

> **상태: LOCKED · v3.0 · 2026-07-18**  
> 전담 사운드 인력 없음 · 라이선스가 확인된 소스 + 편집·레이어·피치 변조로 제작

## 1. 한 문장 오디오 정의

**차갑고 깨끗한 얼음 파열음 위에 낮은 선박 기관음과 따뜻한 항구 신호음이 얹히는 소리.**

오디오는 세계관을 설명하지 않는다. 얼음의 재질, 선박의 존재, 항구에 가까워졌다는 사실을 소리로 보강한다.

---

## 2. 기본 원칙

- 업무 불간섭을 위해 최초 실행 시 소리 기본 꺼짐
- 유저가 켠 설정은 로컬 저장
- 음성 대사·내레이션·캐릭터 보이스 없음
- 공포용 저음 드론과 전쟁용 포성 배제
- 얼음 파괴음은 짧고 선명하게, 선박 소리는 낮고 지속적으로 구성
- 같은 소리 반복을 줄이기 위해 피치·볼륨·시작 위치를 소폭 랜덤화
- 연쇄 파괴 때 개별 소리를 무제한 겹치지 않음

### 감정 대비

| 영역 | 소리 | 감정 |
|---|---|---|
| 얼음바다 | 바람·얼음 마찰·얇은 금속성 고음 | 차가움·거리감 |
| 쇄빙선 | 낮은 디젤 기관·선체 진동 | 안정감·무게감 |
| 파쇄 | 짧은 충격·유리성 파열·파편 | 시원함·성취감 |
| 항구 | 뱃고동·종·나무 부두·따뜻한 화음 | 도착·안도감 |

## 3. 오디오 믹서 구조

```text
Master
├─ Music
├─ SFX_Gameplay
├─ SFX_Ship
├─ Ambience
└─ UI
```

### 권장 상대 레벨

| 그룹 | 기준 |
|---|---:|
| Master Peak | -1 dBFS 이하 |
| SFX_Gameplay | 기준 0 dB |
| SFX_Ship | Gameplay 대비 -4 dB |
| UI | Gameplay 대비 -8 dB |
| Ambience | Gameplay 대비 -12 dB |
| Music | Gameplay 대비 -10 dB |

절대 수치보다 파쇄음이 가장 먼저 들리고 기관음과 음악이 뒤에 남는 관계를 우선한다.

## 5. 빙질별 소리

하나의 기본 타격·파괴 소스를 레이어와 피치로 변형한다.

| 빙질 | 피치 | 추가 레이어 | 인상 |
|---|---:|---|---|
| T1 백빙 | +3~+5 semitone | 얇은 유리 조각 | 가볍고 잘 깨짐 |
| T2 청빙 | 0 | 중간 충격·두꺼운 크랙 | 단단하고 선명함 |
| T3 심빙 | -3~-5 semitone | 짧은 저역 충격 | 무겁고 깊음 |

### 랜덤 변조

- 일반 타격 피치: ±3%
- 일반 타격 볼륨: ±1.5 dB
- 파괴 피치: ±2%
- 좌우 패닝: 얼음 화면 위치에 따라 최대 ±25%

크리티컬 고음과 특수빙 고유음은 피치 랜덤 범위를 절반으로 줄여 인식성을 유지한다.

---

## 6. 연쇄 파괴 믹싱

연쇄가 커질수록 소리를 모두 겹치면 클리핑과 피로가 발생한다.

### 규칙

- 동시 파괴음 보이스 최대 8개
- 0.03초 안에 같은 파괴음이 여러 번 발생하면 가장 큰 1개를 중심으로 묶음
- 9개째부터는 개별 파괴음 대신 연쇄 러시 레이어의 볼륨만 상승
- 연쇄 단계 신호는 최대 5단까지만 피치 상승
- 6단 이상은 같은 최고 피치를 유지하고 볼륨을 더 올리지 않음
- 큰 연쇄 발생 시 Ambience를 0.4초 동안 4 dB 낮춤

### 연쇄 피치

```text
2연쇄  +0
3연쇄  +1 semitone
5연쇄  +2 semitone
10연쇄 +3 semitone
20연쇄 +4 semitone
```

이 소리는 콤보 멜로디가 아니라 파괴가 번지고 있다는 짧은 청각 신호다.

## 7. 60초 흐름

| 구간 | 오디오 |
|---|---|
| 준비 | 낮은 기관음 + 바람, 음악 없음 또는 매우 약함 |
| 카운트다운 | 동일한 틱 3회, 마지막 틱만 약간 높게 |
| 60~6초 | 파쇄음 중심, 기관·바람은 배경 유지 |
| 마지막 5초 | 낮은 펄스 추가, 타격음 피치 변화는 하지 않음 |
| 0초 직전 | 0.1초 Gameplay·Music 정지, 기관음은 아주 작게 유지 |
| 운항 정산 | 화면 정지 후 짧은 해소음 + 항로 전진음 |
| 항해 중 | 기본 무음, 유저가 환경음을 켰을 때만 낮은 기관·바람 |

마지막 5초에 전체 BGM 속도를 올리면 부자연스러우므로 별도 펄스 레이어만 켠다.

## 8. 항구 도착 오디오

항구마다 전용 음악을 만들지 않는다.

1. 성에 프레임 파열
2. 기관 출력이 낮아지는 소리
3. 짧은 뱃고동
4. 화물 내려놓는 소리
5. 따뜻한 2음 도착 화음

북쪽 기지는 다른 항구와 같은 도착 화음을 사용한다. 표준 RC가 안정되고 시간이 남을 때만 선택 엔딩용으로 한 음과 긴 잔향을 추가하며, 별도 소스나 새로운 곡은 만들지 않는다.

---

## 9. BGM

### P0 권장

- 쇄빙 작업용 60~75초 무봉제 루프 1곡
- 템포 70~90 BPM
- 낮은 펄스, 금속성 퍼커션, 얇은 패드
- 명확한 멜로디보다 리듬과 공간감 중심
- 파쇄 효과음을 가리지 않도록 1~4kHz 대역을 비움

### 항해 중

- 기본 무음
- 환경음 옵션을 켠 경우 기관·바람만 재생
- 별도 항해 BGM 없음

### 일정 부족 시

BGM을 가장 먼저 컷하고 파쇄·기관·항구 도착 SFX를 유지한다. 음악이 없어도 게임 정체성이 성립해야 한다.

## 10. 파일 기준

### 원본

- WAV
- 48kHz
- 24bit 권장
- 피크 -1 dBFS 이하
- 루프 파일은 시작·끝 제로 크로싱 확인

### Unity 임포트

- 짧은 UI·파쇄음: Decompress On Load
- 1초 이상 환경음·음악: Vorbis 압축
- 스테레오가 필요 없는 충격음은 Mono
- 화면 위치가 중요한 타격·파괴음만 2D 패닝 적용
- 모든 오디오는 2D로 사용하며 3D 거리 감쇠는 사용하지 않음

## 13. 오디오 승인 기준

- T1·T2·T3를 화면을 보지 않고도 대략 구분할 수 있음
- 20개 연쇄 파괴에서도 클리핑과 불쾌한 고음이 없음
- 파쇄음이 기관·바람·음악보다 먼저 들림
- 소리를 켠 상태에서 60초 반복 플레이해도 같은 샘플 반복이 두드러지지 않음
- 소리 기본 꺼짐과 설정 저장이 동작함
- 항구 도착 시 대사 없이도 평소 파쇄 화면과 다른 안도감을 느낌
- 도킹 상태에서 자동으로 소리가 재생되지 않음
