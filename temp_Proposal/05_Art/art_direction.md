# ICEBREAKER — 아트 디렉션

> **상태: LOCKED · v3.0 · 2026-07-18**  
> 기준 해상도: 확장 720×480 · 도킹 480×56 · 쇄빙 화면 520×376  
> 제작 방향: 픽셀 아트가 아닌 **캐주얼 핸드페인팅 2D + 미형 캐릭터 원화**

## 1. 한 문장 아트 정의

**차가운 청록색 얼음바다를 작은 쇄빙선이 가르고, 멀리 있는 주황색 항구 불빛을 향해 나아가는 캐주얼 2D 게임.**

게임 오브젝트는 작은 화면에서 바로 읽히도록 단순하게 그리고, 캐릭터와 키비주얼에서 원화의 매력을 보여준다.

### 화면이 5초 안에 전달해야 하는 것

1. 화면 아래의 작은 배가 플레이어다.
2. 떠다니는 얼음이 항로를 막고 있다.
3. 멀리 따뜻한 항구가 있다.
4. 배에는 전달할 화물이 있다.

이 네 가지가 보이면 설정 설명을 위한 장식은 추가하지 않는다.

## 2. 공식 시각 레퍼런스와 적용 범위

레퍼런스는 화풍 복제가 아니라 구성·색·가독성 원칙을 확인하는 용도다.

| 레퍼런스 | 가져오는 것 | 가져오지 않는 것 |
|---|---|---|
| [DREDGE 공식 프레스킷](https://www.dredge.game/press-kit) | 작은 배와 먼 섬·등대의 명확한 실루엣, 어두운 바다와 따뜻한 항구 불빛의 대비, 장비를 달아 배가 성장하는 느낌 | 공포 분위기, 기괴한 생물, 저채도 3D 로우폴리 재현 |
| [Spiritfarer 공식 프레스킷](https://thunderlotusgames.com/press-kits/spiritfarer-press-kit/) | 손으로 그린 배와 캐릭터, 따뜻한 조명, 배 자체가 생활 공간이자 캐릭터처럼 느껴지는 구성 | 다수 캐릭터 애니메이션, 건물 배치, 관리 시뮬레이션 밀도 |
| [FAR: Lone Sails 공식 프레스 자료](https://www.far-game.com/for-the-press/) | 독특한 차량의 측면도를 실제 게임 화면으로 사용하는 방식, 부위별 장비가 보이는 실루엣, 작은 주인공과 큰 기계의 대비 | 황폐한 단색 화면, 복잡한 내부 조작, 차량 유지보수 시뮬레이션 |
| [Wildfrost 공식 프레스킷](https://www.wildfrostgame.com/presskit/) | 차가운 소재를 둥글고 또렷하게 단순화한 캐주얼 캐릭터, 작은 크기에서도 읽히는 표정과 형태 | 카드 프레임, 데크빌딩 UI, 과장된 SD 비율의 직접 복제 |

### 레퍼런스 혼합 비율

- 게임 화면의 바다·배 구도: DREDGE 40% + FAR 30%
- 캐릭터와 따뜻한 정서: Spiritfarer 20%
- 얼음·아이콘의 캐주얼 가독성: Wildfrost 10%

이 비율은 작업 우선순위를 설명하기 위한 것이며 특정 작품의 붓터치·캐릭터 디자인을 모사하라는 의미가 아니다.

## 3. 최종 스타일

### 채택

- 캐주얼 핸드페인팅 2D
- 큰 실루엣과 2~3단계 명암
- 부드러운 그라데이션과 제한된 질감
- 색이 있는 외곽선 또는 어두운 남색 외곽선
- 얼음은 투명감, 선박은 불투명하고 단단한 질감
- UI는 현대적인 항해 계기판 느낌

### 배제

- 픽셀 아트
- 도형만 배치한 플랫 벡터
- 사실적인 사진 질감
- 모든 오브젝트에 두꺼운 검정 외곽선
- 스팀펑크 톱니바퀴와 장식 과잉
- 군함·대포 중심의 전쟁 이미지
- 작은 게임 오브젝트에 원화 수준의 세부 묘사
- 화면 전체를 덮는 카드·패널·장식 프레임

### 디테일 배분

- 70%: 한눈에 읽히는 형태
- 20%: 명암과 발광
- 10%: 표면 질감과 작은 장식

68~84px 얼음에서 보이지 않는 디테일은 그리지 않는다.

## 5. 선장 캐릭터

### 역할

선장은 세계관을 설명하는 NPC가 아니라 ICEBREAKER를 대표하는 얼굴이다. 타이틀·선박 정보·정비 화면에서만 사용하고 플레이 화면에 말풍선이나 초상화를 띄우지 않는다.

### 디자인 키워드

- 젊은 쇄빙선 선장
- 단정하고 실용적
- 미형이지만 지나치게 화려하지 않음
- 추위 속에서도 따뜻하고 신뢰할 수 있는 인상
- 현장 경험이 느껴지는 장갑·무전기·공구

### 의상

- 짙은 남색 방한 선장 코트
- 안전 장비를 연상시키는 주황색 목도리
- 청록색 또는 회색 방한 고글
- 두꺼운 장갑과 방수 부츠
- 허리의 무전기·지도 케이스·작은 공구
- 장식용 견장·메달·판타지 문양은 사용하지 않음

### 실루엣

- 코트의 큰 깃과 목도리로 상체 실루엣을 기억하게 만든다.
- 무전기와 고글은 직업을 설명하는 보조 요소다.
- 거대한 무기나 마법 지팡이를 들리지 않는다.
- 표정은 침착한 기본 표정 하나를 P0 기준으로 한다.

### 제작 범위

- P0 캐릭터 1명
- 투명 배경 반신 원화 1장
- 표정 교체 없음
- 립싱크·전신 애니메이션 없음
- 키비주얼은 반신 원화와 게임 에셋을 합성해 제작

## 6. 쇄빙선 ICEBREAKER

### 디자인 목표

실제 작업선처럼 믿을 수 있으면서, 720×480 화면에서 한 번에 기억되는 실루엣이어야 한다.

### 필수 형태

- 위로 들린 두꺼운 쇄빙용 선수
- 짧고 높은 선교
- 화물이 보이는 작은 후방 갑판
- 안테나와 탐조등
- 낮고 넓은 선체
- 창문과 탐조등의 따뜻한 노란빛

### 색

- 선체: 짙은 남색
- 상부 구조물: 서리가 묻은 밝은 회색
- 장비·난간: 안전 주황
- 창문·탐조등: 따뜻한 노랑
- 금속 그림자: 청회색

### 파쇄 장비 표현

전쟁용 대포처럼 보이지 않는 산업 장비로 디자인한다.

1. **주 파쇄기:** 선수에 붙은 충격 발사 장치 또는 기계식 충격기
2. **보조 장비:** 선교 위 소형 보조 발사기
3. **충격 장치:** 선체 측면의 공진판 또는 충격 코일
4. **공통 장비:** 기관실·탐조등·선체 보강

### 정비 화면

- 측면도 한 장을 중심으로 네 장비 영역을 표시한다.
- 12개 노드마다 다른 선박 그림을 만들지 않는다.
- P0에서는 네 장비 영역의 점등·광택·작은 부착물 레이어만 바꾼다.
- 레벨은 UI 숫자와 점으로 표현하고 외형 변화는 브랜치 단위로만 제공한다.

### 플레이 화면

- 뱃머리 일부만 화면 아래 중앙에 보인다.
- 파쇄 장비의 발광부와 주황색 난간이 최소 2곳 보여야 한다.
- 얼음보다 뒤 레이어에 놓고 클릭을 가로막지 않는다.
- 선체 흔들림은 프레임 애니메이션이 아니라 코드 기반 상하 이동으로 처리한다.

## 7. 얼음

### 공통 원칙

- 기본 실루엣 6종을 T1~T3가 공유한다.
- 정사각형·완전한 원형을 피하고 5~8개의 큰 면으로 구성한다.
- 크기 68~84px에서 외곽과 중심 재질이 구분되어야 한다.
- 투명 영역 안에 미세 노이즈를 가득 넣지 않는다.
- 검은 외곽선 대신 재질보다 어두운 청색 테두리를 사용한다.

### T1 백빙

- 둥글고 얇은 모서리
- 흰색과 옅은 청색
- 가장자리 서리, 중심은 비교적 투명
- 내부 기포는 1~2개 큰 형태만 사용

### T2 청빙

- T1보다 각지고 압축된 형태
- 선명한 청록색 중심
- 길게 흐르는 압축선 또는 층리 1개
- 어두운 외곽과 밝은 중심 대비

### T3 심빙

- 두껍고 무게감 있는 모서리
- 짙은 코발트 중심과 청록색 가장자리
- 내부에 큰 어두운 코어 또는 깊은 균열
- 검은 덩어리처럼 보이지 않도록 한쪽 면에 밝은 반사광 유지

### 특수빙

- 결정빙: 중심에서 뻗는 밝은 결정맥 + 작은 마름모 아이콘
- 균열빙: 십자형 또는 방사형 균열 + 불안정한 주황빛 점 1개
- 특수 속성은 기본 얼음 위에 얹는 공용 오버레이로 제작해 모든 티어에 재사용
- P0에서는 결정빙·균열빙만 제작

### 파손 단계

- 무손상
- 균열 1: HP 약 70%
- 균열 2: HP 약 40%
- 균열 3: HP 약 15%
- 파괴: 스프라이트 교체가 아니라 파편과 플래시로 처리

균열 텍스처 3종을 회전·크기 변형해 모든 얼음에 공용 사용한다.

## 8. 바다와 목적지

### 공통 배경

- 하나의 밤하늘·바다 배경을 P0 전체에서 사용한다.
- 바다의 흐름은 가로 줄무늬나 입자보다 큰 명암 덩어리로 표현한다.
- 눈과 안개는 반투명 오버레이와 파티클로 처리한다.
- 화면 아래는 선박 때문에 어둡고, 화면 위 목적지 쪽은 약간 밝게 연다.

### P0 목적지

| 목적지 | 화물 | 시각적 표지 |
|---|---|---|
| 섬마을 | 식료품·우편 | 낮은 지붕, 작은 부두, 여러 개의 따뜻한 창문 |
| 등대항 | 발전기 연료·의약품 | 높은 등대, 연료 탱크, 방파제 |
| 북쪽 기지 | 기계 부품·우편 | 안테나, 모듈형 건물, 강한 탐조등 |

목적지는 실루엣 에셋 1장씩만 제작한다. 항로 진행률에 따라 크기·위치·안개 값을 바꿔 원경·중경·근경·도착 상태를 만든다.

### 항구 도착

- 모든 항구에 같은 부두 전경 레이어를 재사용한다.
- 목적지 실루엣과 불빛 색만 교체한다.
- 사람은 개별 캐릭터가 아니라 화물을 내리는 작은 실루엣 2~3명으로 표현한다.
- 전용 컷신 배경을 추가하지 않는다.

## 9. UI 그래픽

### 방향

- 짙은 남색 반투명 패널
- 밝은 청회색 텍스트
- 주황색은 선택·도착·구매 가능 상태
- 얇은 선과 넓은 여백
- 모서리는 4~6px의 작은 라운드
- 항해 계기판 느낌은 주되 나사·리벳 장식을 반복하지 않음

### 아이콘

- 64×64 기준
- 한 아이콘당 큰 형태 1개 + 보조 형태 최대 2개
- 선 두께는 1× 출력에서 최소 2px
- 흰색·청록색 기본, 활성 상태만 주황색
- 화물 아이콘은 실루엣만 보고 연료·약·우편·식료품·부품을 구분할 수 있어야 함

### 선박 정비 아이콘

P0에서는 13개 노드 전용 아이콘을 만들지 않는다.

- 공통 장비
- 직접 파쇄
- 보조 파쇄
- 연쇄 파괴

네 브랜치 아이콘과 선박 부위 하이라이트를 재사용하고, 개별 노드는 텍스트·레벨 점·잠금 상태로 구분한다.

## 10. VFX와 파괴 피드백

### 타격 피드백 5중 스택

1. 균열 오버레이
2. 피해량에 비례한 파편 수
3. 숫자 팝
4. 확장 화면 한정 미세 진동
5. 재질별 타격음과 파괴음

### 파편

- 기본 파편 실루엣 8종
- 크기·회전·색은 엔진에서 변형
- T1은 밝고 작게, T3는 어둡고 크게
- 결정빙 파편은 같은 실루엣에 밝은 민트색 재질 적용

### 화면 전환

- 마지막 5초: 화면 가장자리 성에
- 시간 종료: 0.3초 정지
- 항로 갱신: 성에 프레임이 갈라져 떨어짐
- 신규 티어: 0.3초 슬로우 + 이름 + 대형 파편
- 항구 도착: 배경 거리 변경 + 불빛 점등 + 화물 아이콘 이동

화면 전체를 흰색으로 번쩍이는 플래시는 0.08초 이하, 불투명도 35% 이하로 제한한다.

## 11. 애니메이션 범위

### 프레임 애니메이션으로 제작

- 타격 플래시 3프레임
- 항구 불빛 점등 2프레임 또는 알파 전환
- 필요 시 선박 탐조등 점멸 2프레임

### 코드·트윈으로 처리

- 선박 상하 흔들림
- 바다 스크롤
- 목적지 크기·안개 전환
- 얼음 부유
- 화물 아이콘 이동
- UI 숫자 팝
- 성에 프레임 파편 이동

### 제작하지 않음

- 캐릭터 전신 애니메이션
- 캐릭터 립싱크
- 선박 프레임별 항해 애니메이션
- 항구별 전용 도착 컷신
- 화물별 하역 애니메이션

## 12. 기술 제작 기준

### 해상도

- 게임용 원본은 목표 표시 크기의 2배로 제작한다.
- 720×480 전체 화면용 원본: 1440×960
- 520×376 쇄빙 화면용 원본: 1040×752
- 68~84px 얼음: 128×128 셀
- 64px UI 아이콘: 128×128 원본

### 파일

- 원본: PSD 또는 Krita, 레이어 유지
- 납품: PNG-24, sRGB
- 투명 에셋은 배경색 잔여 픽셀 없이 알파 정리
- UI와 2D 스프라이트는 밉맵 사용 안 함
- 핸드페인팅 에셋은 Bilinear 필터 기준
- 아틀라스 셀 사이 1× 기준 4px 이상 패딩

### 레이어 이름

- 영문 소문자와 언더스코어 사용
- 예: `ship_hull`, `ship_light`, `ice_t2_shape_03`, `port_lighthouse_glow`
- `final`, `final2`, `수정본` 같은 이름 사용 금지

### 피벗

- 얼음: 중앙
- 선박 측면도: 선체 바닥 중앙
- 플레이용 뱃머리: 화면 아래 중앙
- 목적지: 부두 수평선 중앙
- 파편: 질량 중심

- 차가운 화면 전체에 주황색을 넓게 사용
- 도킹 스트립의 반복 점멸

## 14. 아트 승인 기준

### 68px 얼음 테스트

- T1·T2·T3를 1초 안에 구분할 수 있는가
- 결정빙·균열빙을 색 없이도 패턴으로 구분할 수 있는가
- 균열 1·2·3단계가 실제 게임 크기에서 보이는가

### 720×480 화면 테스트

- 작은 배와 먼 목적지가 동시에 읽히는가
- UI보다 얼음이 먼저 보이는가
- 차가운 바다와 따뜻한 항구의 대비가 유지되는가
- 플레이 영역에 장식 패널이 침범하지 않는가

### 480×56 도킹 테스트

- 선박·목적지·준비 상태가 축소 후에도 구분되는가
- 3초 이상 반복 점멸 없이 준비 완료를 전달하는가
- 업무 화면에서 지나치게 밝거나 시끄럽지 않은가

모든 에셋은 실제 크기 합성본에서 승인한다. 확대 원본만 보고 승인하지 않는다.
