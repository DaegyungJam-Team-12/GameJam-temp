# ICEBREAKER — 아트 디렉션

> **상태: LOCKED · 2026-07-19 · 기준 문서: `new/ICEBREAKER_SPEC_v2.md` v2.6**<br>
> 실행 기준: 쇄빙·정비·운항 현황·정산 960×540 · 축소 게임 바 800×72<br>
> 제작 기준: 확장 화면 1920×1080 · 확장 배경 원본 권장 2560×1440 · 축소 게임 바 1600×144<br>
> 제작 방향: 도트 그래픽이 아닌 **캐주얼 손그림풍 2D + 미형 캐릭터 원화**

> **해상도 기준:** 통합 기획서 v2.6과 동일하게 800×72·960×540을 P0 실행 기준으로 사용한다.

P0는 이번 게임잼에서 반드시 완성할 범위다. P0 창 크기는 800×72와 960×540으로 고정한다. 이후 설정에서 더 큰 창 크기를 지원할 수 있으므로, 아트는 크기별로 다시 그리지 않고 같은 큰 원본을 축소·확대해 쓸 수 있게 제작한다.

## 1. 한 문장 아트 정의

**차가운 청록색 얼음바다를 작은 쇄빙선이 가르고, 멀리 있는 주황색 항구 불빛을 향해 나아가는 캐주얼 2D 게임.**

게임 오브젝트는 작은 화면에서 바로 읽히도록 단순하게 그리고, 캐릭터와 키비주얼에서 원화의 매력을 보여준다.

### 처음 봤을 때 바로 구분되어야 하는 요소

1. 화면 아래의 작은 배가 플레이어다.
2. 떠다니는 얼음이 항로를 막고 있다.
3. 멀리 따뜻한 항구가 있다.
4. 배에는 전달할 화물이 있다.

이 네 가지가 보이면 설정 설명을 위한 장식은 추가하지 않는다.

## 2. 최종 스타일

### 채택

- 캐주얼 손그림풍 2D
- 어두운 청색 바다와 따뜻한 항구 조명
- 큰 실루엣과 2~3단계 명암
- 부드러운 그라데이션과 제한된 질감
- 색이 있는 외곽선 또는 어두운 남색 외곽선
- 얼음은 투명감, 선박은 불투명하고 단단한 질감
- UI는 현대적인 항해 계기판 느낌

### 배제

- 도트 그래픽(픽셀 아트)
- 도형만 배치한 플랫 벡터
- 사실적인 사진 질감
- 모든 오브젝트에 두꺼운 검정 외곽선
- 스팀펑크 톱니바퀴와 장식 과잉
- 군함·대포 중심의 전쟁 이미지
- 작은 게임 오브젝트에 원화 수준의 세부 묘사
- 동전·정비 부품 드롭 에셋 — 재화 표현은 정비 자금 아이콘과 황금색 숫자 스타일뿐이다

### 디테일 배분

- 70%: 한눈에 읽히는 형태
- 20%: 명암과 발광
- 10%: 표면 질감과 작은 장식

작은 화면에서 보이지 않는 디테일은 그리지 않는다.

## 4. 선장 캐릭터

### 역할

선장은 세계관을 설명하는 NPC가 아니라 ICEBREAKER를 대표하는 얼굴이다. 항구 도착과 정비 화면에서 공용으로 사용하고 플레이 화면에 말풍선이나 초상화를 띄우지 않는다.

### 디자인 키워드

- 젊은 쇄빙선 선장
- 단정하고 실용적
- 미형이지만 지나치게 화려하지 않음
- 추위 속에서도 따뜻하고 신뢰할 수 있는 인상
- 현장 경험이 느껴지는 장갑·무전기·공구

### 의상

- 짙은 남색 방한 선장 코트
- 안전 장비를 연상시키는 주황색 목도리
- 청록색 또는 회색 방한 고글
- 두꺼운 장갑과 방수 부츠
- 장식용 견장·메달·판타지 문양은 사용하지 않음

### 제작 범위

- P0 캐릭터 1명 · 반신 원화 1장
- 추가 표정은 P1
- 립싱크·전신 애니메이션 없음
- 키비주얼은 반신 원화와 게임 에셋을 합성해 제작 (일정 여유 시)

## 5. 쇄빙선 ICEBREAKER

### 디자인 목표

실제 작업선처럼 믿을 수 있으면서, 작은 화면에서도 뱃머리와 보조 파쇄 장비가 구분되는 실루엣이어야 한다.

### 필수 형태

- 위로 들린 두꺼운 쇄빙용 선수
- 주 파쇄기: 선수에 붙은 충격 발사 장치 (점등 4단계)
- 보조 파쇄 장비: 대기·충전·발사 준비·발사 4상태, 12칸 충전 고리 분리 가능
- 창문과 탐조등의 따뜻한 노란빛

### 색

- 선체: 짙은 남색
- 상부 구조물: 서리가 묻은 밝은 회색
- 장비·난간: 안전 주황
- 창문·탐조등: 따뜻한 노랑

### 플레이 화면

- 선박 뱃머리와 보조 파쇄 장비는 960×540 게임 영역의 하단 중앙 X 280~680 / Y 405~540에 배치한다.
- 선박·장비를 배경에 합쳐 그리지 않는다. 실제 표시 예정 크기보다 가로·세로 약 2배 큰 원본과 투명 PNG를 별도 레이어·파일로 유지한다.
- 장식이 아니라 실제 시스템 상태를 전달하는 게임 오브젝트다.
- 얼음과 겹쳐도 외곽선이 구분되는 투명 PNG로 제작한다.
- 전쟁용 대포처럼 보이지 않는 산업 장비로 디자인한다.

## 6. 얼음

### 공통 원칙

- 기본 얼음은 T1·T2·T3 각 2형태, 총 6개.
- 원본은 장면마다 256×256 크기의 같은 칸에 제작한다. Unity의 초기 표시 후보는 96~112px이며, 최종 표시 크기와 클릭 판정 반지름은 통합 테스트에서 조정한다.
- 모든 장면은 같은 칸 크기와 중앙 그림 기준점(피벗)을 사용하고, 크기를 줄여도 외곽선과 단계 차이가 뭉개지지 않아야 한다.
- 정사각형·완전한 원형을 피하고 5~8개의 큰 면으로 구성한다.
- 단계별 명도·크기 차이가 작은 화면에서도 식별돼야 한다.
- 검은 외곽선 대신 재질보다 어두운 청색 테두리를 사용한다.

### 단계별 특징

- **T1 백빙:** 둥글고 얇은 모서리, 흰색과 옅은 청색, 가장자리 서리
- **T2 청빙:** 각지고 압축된 형태, 선명한 청록색 중심, 압축선 1개
- **T3 심빙:** 두껍고 무게감 있는 모서리, 짙은 코발트 중심, 한쪽 면에 밝은 반사광 유지

### 특수빙

- 결정빙: 중심에서 뻗는 밝은 결정맥 오버레이 1종
- 균열빙: 십자형 또는 방사형 균열 오버레이 1종
- 기본 얼음 위에 합성 가능한 투명 PNG로 제작해 모든 단계에 재사용

### 파괴 효과

- 기본 파편, 결정 파편, 균열 파동, 큰 붕괴 각 1세트
- 0.45초 안에 보상 숫자를 가리지 않고 사라져야 한다

## 7. 배경과 목적지

### 쇄빙 배경

- 바다·수평선·먼 항구 3레이어를 2560×1440 원본으로 제작한다.
- 1920×1080 제작 합성본과 960×540 화면에 보이도록 잘라 쓴 결과에서 빈 공간이 없어야 한다.
- 바다·수평선·항구를 한 장으로 합치지 않고 레이어를 유지해 비율·위치 조정에 대응한다.
- 화면 아래는 선박 때문에 어둡고, 화면 위 목적지 쪽은 약간 밝게 연다

### 작업 종료용 에셋

- 배경 얼음판 기본·균열·파괴 3상태
- 1.2초 작업 종료 전환 안에 끝나는 파편·성에 연출 (카메라 이동 없이 연결)
- 목적지 도착 시 사용할 항구 이름 표식과 따뜻한 불빛 변화

### P0 목적지

| 목적지 | 화물 | 시각적 표지 |
|---|---|---|
| 섬마을 | 식료품·우편 | 낮은 지붕, 작은 부두, 따뜻한 창문 |
| 등대항 | 발전기 연료·의약품 | 높은 등대, 연료 탱크, 방파제 |
| 북쪽 기지 | 기계 부품·우편 | 안테나, 모듈형 건물, 강한 탐조등 |

항구는 같은 구도에 색·표식만 바꿔 제작량을 줄인다. 목적지별 고유 항구 연출은 일정 여유 시에만 추가한다.

## 8. UI 그래픽

### 방향

- 짙은 남색 반투명 패널, 밝은 청회색 텍스트
- 주황색은 선택·도착·구매 가능 상태
- 얇은 선과 넓은 여백, 모서리는 4~6px의 작은 라운드

### 정비 화면 (고정형 연결 트리)

- 정비 보드 1, **노드 아이콘 13개**, 상태 프레임 3종(잠김·구매 가능·보유), 선택 외곽선 1, 연결선 ON/OFF, 상세 패널, 구매 버튼
- 아이콘 이름과 ID를 C01~H03으로 일치시킨다.
- 모든 노드를 960×540 한 화면에서 식별 가능하게 제작한다.
- 연결선은 아이콘과 이름을 통과하지 않게 노드 뒤에 배치한다.

### 크기 변경 대응

- UI는 전체 화면 한 장이 아니라 패널, 버튼 배경, 아이콘, 진행 바처럼 부품으로 나눠 제작한다.
- 패널과 버튼 배경은 모서리·테두리는 그대로 두고 가운데만 늘려도 깨지지 않도록 영역을 나눈다. Unity에서는 이 방식을 9-slice라고 부른다.
- 텍스트와 숫자를 PNG에 굽지 않는다. 글자는 Unity에서 배치한다.
- 축소 게임 바는 800×72 실행 화면을 기준으로 하되, 원본은 1600×144로 제작한다.
- 확장 UI는 960×540 실행 화면을 기준으로 하되, 원본·목업은 1920×1080으로 제작한다.
- 향후 더 큰 화면용 별도 에셋 세트는 만들지 않는다. 같은 큰 제작 원본을 재사용한다.

### 화물·운항 현황

- 화물 아이콘 5종: 실루엣만 보고 식료품·우편·연료·의약품·부품을 구분할 수 있어야 한다
- 목적지 진행 바 1, 목적지 완료 표식 3
- 경로 선택 UI 없이 현재·다음 목적지 정보만 표시한다

## 9. VFX와 파괴 피드백

1. 균열 오버레이 (3단계)
2. 피해량에 비례한 파편 수
3. 보상 숫자 팝 — 황금색 `+N`, 청백색 치명타, 푸른색 연쇄 3스타일 (외곽선 포함, 0.45초 모션)
4. 확장 화면 한정 미세 진동 (설정에서 끄기 가능)
5. 단계별 타격음과 파괴음

일반 피해 숫자는 표시하지 않는다. 화면 전체를 흰색으로 번쩍이는 플래시는 0.08초 이하, 불투명도 35% 이하로 제한한다.

## 10. 기술 제작 기준

### 해상도와 스케일

| 대상 | 실행 기준 | 제작·납품 기준 |
|---|---:|---:|
| 축소 게임 바 | 800×72 | 1600×144 목업·UI 원본 |
| 확장 화면 | 960×540 | 1920×1080 목업·UI 원본 |
| 확장 배경 | 960×540에 보이도록 잘라 사용 | 2560×1440 레이어 원본 권장 |
| 개별 스프라이트 | Unity에서 크기 조정 | 예상 표시 크기의 가로·세로 약 2배, 같은 칸·그림 기준점 |

800×72와 960×540은 P0 구현 기준이다. 1000×90·1200×675 또는 1200×108·1440×810처럼 더 큰 창 크기는 향후 설정 후보이며, 이번 게임잼에서는 별도 화면·에셋을 만들지 않는다.

### 파일

- 원본: Clip Studio `.clip`. 레이어를 유지한다.
- Unity 전달본: 실제 표시 예정 크기보다 가로·세로 약 2배 큰 투명 PNG(PNG-24, sRGB)를 기준본으로 제공한다.
- 실제 게임 크기 파일은 전체 화면 승인용 합성본만 필수다. 개별 에셋의 실제 크기 파일은 개발팀 요청이 있을 때만 추가한다.
- 파일명은 `범주_ID_상태` 형식으로 통일 (영문 소문자·언더스코어)
- `final`, `final2`, `수정본` 같은 이름 사용 금지
- 투명 에셋은 가장자리에 배경색 잔여 픽셀이 남지 않게 정리
- 애니메이션 시트의 각 칸 사이는 실제 게임 크기로 줄였을 때 4px 이상 띄움

### 그림 기준점(피벗)

- 얼음: 중앙
- 플레이용 뱃머리: 화면 아래 중앙
- 보조 파쇄 장비: 같은 기준점과 크기 유지
- 파편: 질량 중심

- 목적지마다 새 배경 원화 제작 (색·표식 변주로 해결)
- 작은 얼음에 눈송이·기포·균열을 모두 넣는 과밀 묘사
- T3를 단순히 T1의 어두운 색상 교체로 처리
- 차가운 화면 전체에 주황색을 넓게 사용
- 축소 게임 바의 반복 점멸

## 12. 아트 승인 기준

### 얼음 테스트

- T1·T2·T3를 실제 게임 크기에서 1초 안에 구분할 수 있는가
- 결정빙·균열빙을 색 없이도 패턴으로 구분할 수 있는가

### 960×540 확장 화면 테스트

- 작은 배와 먼 목적지가 동시에 읽히는가
- 상단 오버레이(자금·타이머·설정)보다 얼음이 먼저 보이는가
- 차가운 바다와 따뜻한 항구의 대비가 유지되는가
- 정비 트리 13개 노드가 한 화면에서 모두 식별되는가

### 800×72 축소 바 테스트

- 정비 자금·목적지 진행·버튼들이 축소 후에도 구분되는가
- 반복 점멸 없이 쇄빙 준비 완료를 전달하는가
- 업무 화면에서 지나치게 밝거나 시끄럽지 않은가

모든 에셋은 960×540·800×72 실제 게임 크기 합성본에서 승인한다. 큰 제작 원본만 보고 승인하지 않는다. UI와 배경은 개발팀이 더 큰 창 크기 미리보기에서도 잘림·빈 공간·과도한 흐림이 없는지 추가 확인한다.
