## 1. 제작 원칙

- 한 번 그린 에셋을 크기·색·표식 변주로 재사용한다.
- P0는 이번 게임잼에서 반드시 완성할 범위다. 실행 기준은 축소 게임 바 800×72, 확장 화면 960×540이다.
- 원본은 Clip Studio `.clip`로 보관하고 Unity 전달본은 실제 표시 예정 크기보다 가로·세로 약 2배 큰 투명 PNG로 납품한다.
- 파일명은 `범주_ID_상태` 형식으로 통일한다.
- 확장 화면 목업·UI 원본은 1920×1080, 축소 게임 바 원본은 1600×144로 제작한다.
- 확장 배경 원본은 2560×1440을 권장하고 바다·수평선·항구 레이어를 유지한다.
- 모든 에셋은 960×540과 800×72 실제 게임 크기 합성 화면에서 확인한 뒤 완료 처리한다.
- 향후 더 큰 창 크기용 별도 에셋 세트는 만들지 않고 같은 큰 제작 원본을 재사용한다.
- 동전·정비 부품 드롭 에셋은 제작하지 않는다. 재화 표현은 정비 자금 아이콘과 황금색 숫자 스타일뿐이다.
- 커서 범위 점선 원은 현재 코드에서 절차적으로 그린다. P0 아티스트 납품 에셋으로 요구하지 않으며, 교체가 필요할 때만 투명 PNG 256×256 원형 링 1장을 추가한다.

- Unity 전달본은 투명 PNG로 제공하며, 오디오 제작·편집은 담당 범위에서 제외

## 6. 최종 납품 체크리스트

- 원본 레이어 파일(`.clip`)과 실제 표시 예정 크기보다 가로·세로 약 2배 큰 PNG 출력물이 모두 있음
- 투명 PNG 가장자리에 배경색 잔여 픽셀이 없음
- 960×540·800×72 실제 게임 크기에서 확인함
- UI가 한 장으로 합쳐지지 않고 패널·버튼·아이콘·진행 바로 분리되어 있음
- 패널·버튼은 모서리를 유지한 채 가운데만 늘릴 수 있고 텍스트가 PNG에 포함되지 않음
- UI 아이콘이 작은 표시 크기에서도 구분됨
- 노드 아이콘 14개가 C01~H03·D04 ID와 일치함
- 파일명이 `범주_ID_상태` 규칙을 따름
- Unity 테스트 캡처 960×540과 800×72를 저장함
- 개발팀이 1.25×·1.5× 미리보기에서 잘림·빈 공간·과도한 흐림을 확인함
- 사용한 외부 브러시·텍스처의 상업 이용 가능 여부를 기록함
