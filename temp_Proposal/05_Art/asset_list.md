## 1. 우선순위 정의

| 등급 | 의미 | 컷 가능 여부 |
|---|---|---|
| P0-A | 첫 플레이어블과 파쇄 손맛에 필수 | 컷 금지 |
| P0-B | 쇄빙선·화물·항구 콘셉트 전달에 필수 | 기능 축소 가능, 삭제 금지 |
| P0-C | 발표·스토어·PPT 완성도 향상 | 일정 부족 시 컷 가능 |

### 제작 원칙

- 한 번 그린 에셋을 크기·색·안개·알파로 재사용한다.
- 게임 표시 크기의 2배 원본으로 제작한다.
- 캐릭터·선박·얼음·항구를 각각 다른 사람이 그린 것처럼 보이지 않게 같은 색과 외곽선 규칙을 적용한다.
- 모든 에셋은 720×480 실제 합성 화면에서 확인한 뒤 완료 처리한다.

## 13. 일정 부족 시 컷 순서

다음 순서대로 줄인다.

1. 키비주얼 전용 보정 — 게임 에셋 단순 합성으로 대체
2. 워드마크 장식 — 일반 굵은 산세리프 로고로 대체
3. 하역 작업자 실루엣 — 화물 아이콘 이동만 유지
4. 장비 부위의 실제 외형 변화 — 발광 하이라이트만 유지
5. 눈·안개 전용 오버레이 — 파티클과 단색 알파로 대체
6. 얼음 실루엣 6종을 4종으로 축소

### 절대 컷하지 않음

- T1·T2·T3의 구분
- 결정빙·균열빙 오버레이
- 균열 3단계와 파편
- 플레이용 쇄빙선 뱃머리
- 차가운 바다와 따뜻한 목적지 불빛
- 화물 아이콘
- 항로 지도 선박·목적지 표시

## 14. 최종 납품 체크리스트

- 원본 레이어 파일과 PNG 출력물이 모두 있음
- 투명 PNG 가장자리에 배경색 잔여 픽셀이 없음
- 1× 실제 크기에서 확인함
- UI 아이콘이 16~24px 표시에서도 구분됨
- 선박 장비 하이라이트가 측면도와 정확히 맞음
- 목적지 실루엣이 원경·중경·근경 크기에서 모두 읽힘
- 아틀라스 셀에 4px 이상 패딩이 있음
- 파일명이 영문 소문자·언더스코어 규칙을 따름
- Unity 테스트 캡처 720×480과 480×56을 저장함
- 사용한 외부 브러시·텍스처의 상업 이용 가능 여부를 기록함
