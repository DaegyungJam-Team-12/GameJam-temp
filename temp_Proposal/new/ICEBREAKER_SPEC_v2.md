# ICEBREAKER — 통합 기획서 v2.6

> 팀 내부 개발 기준 문서  
> 플랫폼: Windows · Unity 6 · 게임잼 P0 기준  
> 기준일: 2026-07-19

이 문서는 ICEBREAKER의 시스템, UI·UX, 밸런스, 개발 분담을 한곳에서 관리하는 최상위 기준이다. PPT는 이 문서를 팀이 빠르게 이해하도록 요약하며, 구현 중 해석이 충돌하면 이 문서를 우선한다.

## 0. 이번 버전에서 잠근 것

- 60초 `쇄빙 작업`과 30초 `항해`를 반복한다.
- 한 화면에 최대 20개의 얼음이 떠다니며, 부서진 얼음은 작업 시간 안에 다시 생성된다.
- 얼음을 파괴한 순간 `정비 자금`과 목적지 진행을 얻는다.
- 정비 자금은 한 종류만 사용하며 동전이나 부품을 줍는 과정은 없다.
- 작업 종료 뒤에는 매번 `자금 정산` 화면을 띄운다. 이미 지급된 이번 작업 자금과 성과를 요약하며 추가 지급은 하지 않는다.
- 쇄빙 화면은 720×480 전체를 게임 영역으로 사용하고, 상단에는 정비 자금·타이머·설정만 띄운다.
- 축소 게임 바는 정비 자금·목적지 진행·정비·운항 현황·쇄빙 시작·설정을 한곳에 제공하는 메인 진입점이다.
- 선박 정비는 13개 노드를 한 화면에 모두 표시하는 고정형 연결 트리로 구성한다.
- 목적지 진행과 현재 얼음 단계는 쇄빙 중 숨기고, 축소 화면과 운항 현황 화면에서 확인한다.
- 운항 현황은 항해 중 필요할 때 여는 보조 정보 화면이며 쇄빙 화면이나 자금 정산 화면을 대체하지 않는다.
- 자동 공격과 방치 수익은 없다. 보조 파쇄는 사용자가 입력하는 동안에만 충전되고 작동한다.
- 필수 완료 범위는 세 목적지 도달까지다. 별도 엔딩은 제출 후보가 안정된 뒤에만 검토한다.
- 비대면 개발은 7월 19일 일요일부터 22일 수요일까지 진행하며, 수요일까지 처음부터 끝까지 플레이 가능한 Windows 빌드를 만든다.
- 7월 23일 목요일과 24일 금요일 대면 해커톤에서는 새 핵심 시스템보다 밸런스·연출·아트·오디오·제출 안정성을 보완한다.
- 민준은 진행·경제·정비·저장·최종 통합, 시연은 얼음 파쇄 게임플레이, 정환은 HUD·피드백·정산 표시·QA를 담당한다.
- 게임 수치 변경은 민준, 파괴 판정은 시연, 전달받은 값의 화면 표시는 정환이 맡아 같은 상태를 여러 곳에서 계산하지 않는다.
- 얼음은 해금 상태별 고정 확률표로 생성한다. 기본 T1 100% → 청빙 대응 후 T1 60%·T2 40% → 심빙 대응 후 T1 20%·T2 45%·T3 35%.
- 얼음 스프라이트는 68~84px, 클릭 판정은 얼음 중심 기준 반지름 44px 원형, 생성 시 중심 사이 최소 거리는 88px다.
- 최초 실행은 항해 대기 없이 쇄빙 준비 완료 상태에서 시작한다.
- 앱 종료 중에도 다음 쇄빙 가능 시각은 절대 시각 기준으로 진행하며, 자금과 목적지 진행은 변하지 않는다.
- 설정 화면을 여는 동안 게임을 일시정지한다. 설정 항목은 마스터 볼륨·화면 진동·게임 종료 3개뿐이다.
- 별도 타이틀·메인 화면은 없다. 실행하면 바로 480×56 축소 런처가 뜬다.

## 1. 게임 설명과 플레이어 역할

### 한 줄 설명

**작업 표시줄 위의 쇄빙선을 열어 60초 동안 얼음을 부수고, 정비 자금으로 배를 강화해 세 목적지까지 가는 PC 게임.**

### 플레이어 역할

플레이어는 겨울 항로에 남은 마지막 보급 쇄빙선 `ICEBREAKER`의 선장이다. 배에는 식료품, 우편, 발전기 연료, 의약품, 기계 부품이 실려 있다. 게임은 이 배경을 긴 설명이나 대화로 전달하지 않는다.

화면에서 다음 요소만 반복해 보여 준다.

- 작업 표시줄 위에 머무는 작은 선박 게임 바
- 멀리 보이는 다음 목적지의 불빛
- 배에 적재된 화물 이름
- 얼음이 부서질수록 앞으로 나아가는 배
- 축소 게임 바에서 바로 여는 선박 정비와 운항 현황 화면

### 목표

`출발항 → 섬마을 → 등대항 → 북쪽 기지` 순서로 이동한다. 세 목적지 목표는 각각 120, 600, 2,400이다.

## 2. 전체 반복 구조

```text
항해 및 정비 대기
→ 다음 쇄빙까지 30초
→ 축소 화면의 쇄빙 시작 버튼 활성화
→ 사용자가 쇄빙 시작
→ 3·2·1 카운트다운
→ 쇄빙 작업 60초
→ 작업 종료 1.2초
→ 자금 정산 화면(최소 1.2초·최대 4초)
→ 축소 화면 복귀·다음 항해 30초
```

### 항해

- 축소 화면 480×56으로 작업 표시줄 바로 위에 머문다.
- 정비 자금, 현재 목적지 진행, 정비, 운항 현황, 쇄빙 시작, 설정을 표시한다.
- 정비와 운항 현황 버튼은 해당 720×480 화면을 바로 연다. 별도의 중간 관리 화면은 만들지 않는다.
- 30초 동안 쇄빙 시작 버튼은 `다음 쇄빙 00:24`처럼 남은 시간을 표시하며 비활성화한다.
- 30초가 끝나면 자동으로 창을 펼치거나 포커스를 가져오지 않고 쇄빙 시작 버튼만 활성화한다.
- 쇄빙 시작 전까지 정비와 운항 현황을 계속 이용할 수 있다.
- 최초 실행은 항해 대기 없이 쇄빙 준비 완료 상태에서 시작한다.
- 앱을 종료해도 다음 쇄빙 가능 시각은 절대 시각으로 저장되어 계속 진행된다. 재실행 시 시각이 지났으면 바로 쇄빙 준비 완료 상태다. 앱 종료 중 자금·목적지 진행은 변하지 않는다.

### 쇄빙 작업

- 사용자가 축소 화면 또는 정비·운항 현황 화면의 `쇄빙 시작` 버튼으로 직접 시작한다.
- 작업 시간은 항상 60초다.
- 클릭과 누르고 있기로 얼음을 공격한다.
- 720×480 전체를 게임 영역으로 사용하고 정비 자금·타이머·설정만 상단에 띄운다.
- 작업 중에는 선박 정비, 운항 현황, 접기 메뉴를 화면에 표시하지 않는다. 설정만 상단 오른쪽에서 사용할 수 있다.
- 목적지 목표를 도중에 달성해도 남은 시간 동안 정비 자금을 계속 획득한다.

### 작업 종료와 자금 정산

- 타이머가 0이 되면 입력, 얼음 생성, 진행 중 피해를 즉시 차단한다.
- 0.2초간 정지한 뒤 남은 1.0초 동안 배경 얼음판의 균열·파괴와 정산 화면 전환을 진행한다.
- 자금 정산 화면은 매 쇄빙 작업 뒤 반드시 표시한다.
- 정산 화면에는 `획득 정비 자금`, `파괴한 얼음 수`, `목적지 진행 증가량`을 표시한다.
- 표시 값은 이번 작업에서 승인된 `IceDestroyedEvent`의 합계다.
- 정비 자금은 이미 파괴 순간 반영됐으므로 정산 화면에서는 다시 증가시키지 않는다.
- 정산 화면은 최소 1.2초 동안 유지한다. 이후 버튼·화면 클릭·Enter·Space 입력으로 넘길 수 있다.
- 입력하지 않으면 정산 화면이 열린 시점부터 4초 뒤 자동으로 항해로 전환한다.
- 30초 항해 시간은 정산 화면이 닫힌 뒤 시작한다.
- 목적지에 도달했을 때는 정산 화면 뒤 항구 이름과 1~2초 도착 연출을 추가한다.

## 3. 얼음 생성·피해·파괴 규칙

### 기본 규칙

- 확장 화면에는 최대 20개의 얼음이 동시에 존재한다.
- 얼음은 서로 겹치지 않는 자유 위치에 생성한다.
- 얼음 스프라이트는 68~84px, 클릭 판정은 얼음 중심 기준 반지름 44px 원형이다.
- 생성 시 얼음 중심 사이 최소 거리 88px를 우선 적용한다. 30회 시도 안에 자리를 찾지 못하면 거리 조건을 완화해 배치한다.
- 파괴 후 0.12초 공백과 0.18초 생성 연출을 거쳐 다시 등장한다.
- 연쇄로 여러 개가 파괴되면 0.03초 간격으로 순서대로 생성한다.
- 다시 생성된 얼음은 0.25초 동안 연쇄 피해를 받지 않는다.

### 기본 전투 수치 v0

| 항목 | 값 |
|---|---:|
| 기본 클릭 피해 | 1 |
| 누르고 있기 | 초당 5회 |
| 치명타 | 5% · 피해 ×3 |
| T1 백빙 HP | 10 |
| T2 청빙 HP | 60 |
| T3 심빙 HP | 360 |

일반 피해 숫자는 표시하지 않는다. 치명타와 연쇄만 표시해 정비 자금 표시와 겹치지 않게 한다.

### 얼음 단계

| 개발 표기 | 화면 이름 | HP 배수 | 정비 자금 | 등장 조건 |
|---|---|---:|---:|---|
| T1 | 백빙 | ×1 | 10 | 처음부터 |
| T2 | 청빙 | ×6 | 80 | 청빙 대응 구매 후 |
| T3 | 심빙 | ×36 | 700 | 심빙 대응 구매 후 |

상위 얼음은 HP 증가보다 정비 자금 증가가 크다. 새 얼음을 안정적으로 부술 수 있게 되는 순간부터 자금 효율이 높아지는 것이 성장 보상이다.

### 얼음 단계 생성 확률

| 해금 상태 | 생성 확률 |
|---|---|
| 기본 | T1 100% |
| 청빙 대응 구매 후 | T1 60% · T2 40% |
| 심빙 대응 구매 후 | T1 20% · T2 45% · T3 35% |

- 재생성마다 독립적으로 판정한다.
- 특수빙 여부는 단계 결정 후 별도로 판정한다.
- 이 표는 v0 초기값이며 화요일 통합본에서 1차 조정한다.

### 특수빙

| 종류 | 등장 | 내구도 | 효과 | 자금 |
|---|---:|---:|---|---:|
| 결정빙 | 2.5% · T2 이상 | 기본과 동일 | 파괴 시 파편 5개가 더 낮은 단계 얼음을 찾아 즉시 파괴한다. 대상이 없으면 소멸한다. | 해당 단계 ×4 |
| 균열빙 | 2% · 전 단계 | 기본 ×0.6 | 파괴 시 반경 120px에 현재 클릭 피해 ×3, 최대 3단 연쇄 | 해당 단계 ×1 |

- 특수빙은 한 화면에 최대 2개만 존재한다.
- 특수 효과로 파괴된 얼음도 각각 정비 자금과 목적지 진행을 얻는다.
- 같은 얼음에 직접·보조·연쇄 피해가 동시에 들어와도 파괴 처리는 한 번만 발생한다.

## 4. 정비 자금과 목적지 진행

### 재화의 의미

정비 자금은 얼음 속에서 발견되는 물건이 아니다. 막힌 항로의 얼음을 제거한 작업량에 따라 확보되는 선박 유지 비용이다. 이 설정을 별도의 이야기 창으로 설명하지 않고 `얼음 파괴 → +금액 → 선박 정비`의 연결로 이해시킨다.

### 기본 지급액

| 얼음 | 기본 지급액 | 결정빙 | 균열빙 |
|---|---:|---:|---:|
| T1 백빙 | 10 | 등장하지 않음 | 10 |
| T2 청빙 | 80 | 320 | 80 |
| T3 심빙 | 700 | 2,800 | 700 |

### 계산식

```text
정비 자금 증가 = 내림(
  단계 기본 지급액
  × 특수빙 배수
  × (1 + 정비 효율 단계 × 0.10)
)
```

- 정비 효율은 같은 종류끼리 더한다.
- 결정빙 배수와 정비 효율은 서로 곱한다.
- 현재 자금은 정수로 저장하며 0보다 작아질 수 없다.

### 목적지 진행

- 얼음 하나를 파괴할 때마다 현재 목적지 진행도가 `+1` 증가한다.
- 얼음 단계, 특수빙, 파괴 원인은 목적지 진행 증가량에 영향을 주지 않는다.
- 현재 목적지 목표에 도달하면 이후 파괴는 자금만 지급한다.
- 초과분은 다음 목적지로 넘기지 않는다.
- 목적지 도착 여부는 작업 종료 때 확인한다.
- 목적지는 T2·T3 얼음을 해금하지 않는다. 새 얼음 단계는 선박 정비의 `청빙 대응`, `심빙 대응`만 해금한다.

### 파괴 이벤트 계약

실제 이벤트명·필드·ID·좌표·발행 순서는 [`07_Technical/implementation_lock.md` 3장](../07_Technical/implementation_lock.md#3-공용-이벤트-계약)을 단일 출처로 사용한다. 파쇄 시스템의 `IceDestroyedEvent`를 받은 진행 시스템이 다음을 한 번에 처리한다.

1. `iceInstanceId`가 이미 처리됐는지 확인한다.
2. 단계와 특수빙에 따른 자금을 계산한다.
3. 보유 정비 자금을 즉시 증가시킨다.
4. 현재 목적지 진행도가 목표 미만이면 진행도를 1 증가시킨다.
5. UI에 파괴 위치와 지급액을 전달한다.
6. 저장 상태를 변경됨으로 표시한다.

`StageEnded`에는 자금 증가값을 넣지 않는다.

작업별 요약을 위해 `stageEarnedFunds`, `stageDestroyedCount`, `stageDestinationProgressGain`을 별도로 누적한다. 세 값은 자금 정산 화면에서 읽기만 하고 다음 쇄빙 작업 시작 시 0으로 초기화한다.

```text
SettlementSummary
- earnedFunds: long
- destroyedCount: int
- destinationProgressGain: int
- reachedDestination: bool
- destinationId: string 또는 null
```

- 진행 시스템은 타이머가 끝나는 순간 누적값을 읽기 전용 `SettlementSummary`로 고정한다.
- `SettlementReady(summary)`를 발생시키며 UI는 전달받은 값을 표시만 한다.
- `destroyedCount`는 직접·보조·연쇄를 포함해 승인된 고유 `IceDestroyedEvent` 수다.
- `destinationProgressGain`은 목표 초과분을 제외한 실제 목적지 진행 증가량이다. 목표 달성 뒤에는 `0`이 될 수 있다.
- `reachedDestination`이 참이면 `destinationId`로 정산 패널에 목적지 이름을 표시한다.
- 정산 UI는 이 데이터를 근거로 자금·목적지 진행·저장 상태를 다시 변경하지 않는다.

### 시간 경계와 저장

- 얼음 HP가 0이 된 시점이 타이머 0 이전일 때만 파괴 이벤트를 인정한다.
- 타이머 0 이후 도착한 투사체와 연쇄 피해는 취소한다.
- 자금과 목적지 진행은 메모리에서 즉시 변경한다.
- 저장 파일 쓰기는 최대 1초 동안 묶어서 처리한다.
- 작업 종료, 정비 구매, 앱 정상 종료 때는 대기 중인 저장을 즉시 완료한다.
- 다음 쇄빙 가능 시각은 절대 시각으로 저장한다. 앱 종료 중에도 시간은 진행되지만 자금과 목적지 진행은 변하지 않는다.

## 5. 목적지 진행과 운항 현황

| 구간 | 목적지 | 화물 | 목표 |
|---|---|---|---:|
| 1 | 섬마을 | 식료품·우편 | 120 |
| 2 | 등대항 | 발전기 연료·의약품 | 600 |
| 3 | 북쪽 기지 | 기계 부품·우편 | 2,400 |

- 쇄빙 중 목적지 진행 수치를 표시하지 않는다.
- 축소 화면에서 현재 목적지와 진행도를 항상 간단히 표시한다.
- 운항 현황 화면에서는 현재 목적지, 진행도, 운송 화물, 이후 목적지 순서만 확인한다. 경로 선택이나 배 조종은 없다.
- 운항 현황은 항해 중 필요할 때만 직접 여는 보조 화면이다. 기본 플레이 화면은 쇄빙 작업이며 작업 종료 뒤에는 반드시 자금 정산을 표시한다.
- 자금 정산 화면에서 이번 작업의 목적지 진행 증가량을 확인하고, 이후 축소 화면의 진행도가 짧게 증가한다.
- 북쪽 기지 도착이 P0 완료 조건이다.
- 별도 엔딩 장면은 12일차 제출 후보가 안정됐을 때만 마지막 순서로 검토한다.

### 화면 표시 규칙

- 정비 화면은 13개 강화가 한 화면에 모두 보이는 고정형 연결 트리로 구성한다.
- `공통 장비 / 직접 파쇄 / 보조 파쇄 / 연쇄 파괴`는 별도 탭이 아니라 한 보드 안의 가지 이름으로만 표시한다.
- 계통별 탭, 스크롤, 확대·축소, 드래그 이동은 사용하지 않는다.
- 연결선은 선행 조건을 표현하며 아이콘과 이름을 통과하지 않게 노드 뒤에 배치한다.
- 노드에는 아이콘과 현재 단계만 표시한다. 노드를 선택하면 하단 상세 패널에 이름, 현재 단계, 현재 효과, 다음 효과, 다음 비용, 구매 버튼을 표시한다.
- 상태는 `보유 / 구매 가능 / 잠김` 세 가지로 제한하며, 선택한 노드는 현재 상태 위에 이중 청색 외곽선을 추가한다.
- 구매 완료 시 해당 노드와 다음에 열리는 연결선이 점등된다.
- 선행 조건을 만족하지 않은 노드는 자물쇠와 필요한 선행 노드를 표시한다.
- 구매 화면은 효과나 비용을 다시 계산하지 않고 `MaintenanceDefinition`과 노드 상태 데이터를 그대로 표시한다.

### 선박 정비·운항 현황 화면 720×480

- 축소 화면의 정비 또는 운항 현황 버튼을 누르면 선택한 화면을 바로 연다. 빈 중간 화면은 없다.
- 두 화면은 공통 헤더를 사용한다.

```text
[정비] [운항 현황]        [정비 자금] [다음 쇄빙 00:24] [설정] [접기]
```

- 정비 버튼으로 열면 정비 탭, 운항 현황 버튼으로 열면 운항 현황 탭이 처음부터 선택돼 있다.
- 공통 헤더에서 두 탭을 서로 전환할 수 있다.
- 정비 화면 제목은 `선박 정비·강화`로 쓰고, 버튼과 탭은 짧게 `정비`로 표시한다.
- 운항 현황은 현재 목적지, 진행도, 운송 화물, 이후 목적지만 표시하며 경로 선택 기능은 제공하지 않는다.
- 운항 현황은 쇄빙 작업과 자금 정산을 대신하지 않으며, 항해 중 목적지 정보를 확인하는 용도로만 사용한다.
- 헤더의 카운트다운은 30초가 끝나면 `쇄빙 시작`으로 바뀐다. 사용자는 화면을 접지 않고도 쇄빙을 시작할 수 있다.
- 얼음과 파쇄 입력은 표시하지 않으며 쇄빙 작업 화면과 시각적으로 구분한다.
- 자동으로 열거나 입력 포커스를 강제로 가져오지 않는다.
- 접기를 누르면 480×56 축소 화면으로 돌아간다.
- 쇄빙 시작 시 정비·운항 현황 화면을 닫고 720×480 쇄빙 화면으로 전환한다.

### 얼음과 선박 안전 영역

- 정비 자금 보호 영역: X 0~216 / Y 0~60
- 타이머 보호 영역: X 284~436 / Y 0~60
- 설정 보호 영역: X 656~720 / Y 0~60
- 선박·보조 파쇄 장비 영역: X 220~520 / Y 360~480
- 얼음 배치 외곽 여백: 16px
- 최대 20개의 얼음은 보호 영역과 외곽 여백을 침범하지 않게 생성한다.
- 보상 숫자는 파괴 위치를 기준으로 하되 X 24~696 / Y 60~344 안으로 보정한다.
- 화면 하단의 선박과 보조 파쇄 장비는 장식이 아니라 실제 시스템 상태를 전달하는 게임 오브젝트다.

### 보조 파쇄 상태

별도 정보 패널이나 분수형 충전 숫자는 사용하지 않는다. 선박에 장착된 장비 자체로 상태를 전달한다.

| 상태 | 화면 표현 |
|---|---|
| 대기 | 조명 꺼짐, 충전 링 비어 있음 |
| 충전 중 | 유효한 직접 입력에 따라 12칸 충전 링이 차오름 |
| 발사 준비 | 충전 링 완성, 장비가 짧게 맥동 |
| 발사 | 0.15~0.25초 궤적·반동·발광 연출 |

### 보상 표시

- 저장 파일에서 첫 파괴 여부를 확인한다.
- 첫 파괴는 `정비 자금 +10`처럼 전체 명칭을 표시한다.
- 이후에는 `+10`, `+80`, `+700`처럼 금액만 표시한다.
- 색상은 황금색, 표시 시간은 0.45초다.
- 실제 자금은 즉시 증가하고 상단 왼쪽 보유액 숫자만 0.2초 동안 따라 올라간다.
- 0.1초 안에 같은 연쇄·보조 공격으로 3개 이상 파괴되면 중심 위치에 합계 하나만 표시한다.
- 1,000 이상은 `1.2K`, `2.8K` 형식으로 줄여 표시한다.

### 자금 정산 화면 720×480

| 요소 | 위치·크기 | 내용 |
|---|---:|---|
| 배경 딤 | X 0 / Y 0 / 720×480 | 60% 어두운 청흑색, 종료 시점의 균열 난 얼음판 유지 |
| 정산 패널 | X 120 / Y 64 / 480×352 | 중앙 정산 정보 |
| 제목 | Y 88 | `쇄빙 작업 정산` |
| 핵심 보상 | Y 132~220 | `획득 정비 자금 +1,240` |
| 보조 정보 | Y 236~284 | `파괴한 얼음 37개`, `목적지 진행 +37` |
| 반영 상태 | Y 292 | `정비 자금 반영 완료` |
| 계속 버튼 | X 230 / Y 336 / 260×48 | `항해 계속` |
| 자동 진행 안내 | 계속 버튼 아래 | `4초 뒤 자동 항해` |

- 작업 종료 연출 뒤 전체 720×480 화면을 어둡게 하고 중앙에 정산 패널을 표시한다.
- `획득 정비 자금`을 가장 크게, 파괴 수와 실제 목적지 진행 증가량을 보조 정보로 둔다.
- 목적지 목표 달성 뒤의 추가 파괴는 자금과 파괴 수에는 포함하지만 목적지 진행 증가량에는 포함하지 않는다.
- `정비 자금 반영 완료`를 표시해 종료 시 다시 지급되는 금액이 아님을 알린다.
- 획득액은 `K`로 줄이지 않고 쉼표가 포함된 정확한 정수로 표시한다.
- 처음 1.2초 동안은 계속 버튼을 비활성화하고 화면을 건너뛸 수 없다.
- 1.2초 이후 버튼·화면 클릭·Enter·Space 입력으로 항해를 계속할 수 있다.
- 입력이 없으면 정산 화면이 열린 시점부터 4초 뒤 자동으로 닫힌다. 안내 문구는 `4 → 3 → 2 → 1`로 갱신한다.
- 정산 화면에서는 상단 설정을 포함한 모든 UI와 얼음 입력을 잠근다.
- 목적지에 도달한 경우 같은 패널 상단에 `목적지 도달` 배지와 항구 이름을 추가하고, 패널이 닫힌 뒤 1~2초 도착 연출을 재생한다.

### 피드백 우선순위

1. 얼음 파괴와 정비 자금
2. 치명타
3. 연쇄 수와 범위
4. 보조 파쇄 충전·발사

보조 파쇄는 선수 장비의 발광, 12칸 충전 링, 맥동, 발사 방향으로 보여 준다.

### 설정 화면

- 설정 항목은 세 개뿐이다: 마스터 볼륨 슬라이더(기본 0 · 최초 실행 음소거), 화면 진동 켬/끔, 게임 종료.
- 중앙 소형 모달 320×240으로 띄우며 별도 전체 화면을 만들지 않는다.
- 설정 화면이 열려 있는 동안 게임을 일시정지한다. 쇄빙 타이머·항해 타이머·얼음 생성·진행 중 피해·보조 파쇄 충전을 멈추고 얼음 입력을 차단한다.
- 설정을 닫으면 멈춘 시점부터 그대로 재개한다.
- 자금 정산 화면에서는 설정을 열 수 없다.
- 저장 초기화 항목은 두지 않는다.

### 접근성과 업무 방해 방지

- 최초 실행은 음소거다.
- 미세 화면 진동은 쇄빙 작업 화면에서만 사용하며 설정에서 끌 수 있다.
- 색상 외에도 `+` 기호, 숫자 크기, 장비 발광 단계로 상태를 구분한다.
- 창 확장·축소 애니메이션은 300ms 이내로 한다.

## 8. 아트·오디오 제작 기준

### 아트

- 캐주얼 손그림풍, 어두운 청색 바다와 따뜻한 항구 조명을 사용한다.
- 도트 그래픽은 사용하지 않는다.
- 선박은 작은 화면에서도 뱃머리와 보조 파쇄 장비가 구분돼야 한다.
- 동전이나 정비 부품 드롭 에셋은 제작하지 않는다.
- 필요한 재화 표현은 정비 자금 아이콘과 황금색 숫자 스타일뿐이다.
- 쇄빙 작업 배경은 720×480 전체 화면을 기준으로 제작한다.

### 작업 종료용 에셋

- 배경 얼음판 기본, 균열, 파괴 3상태
- 1.2초 작업 종료 전환 안에 끝나는 파편·성에 연출
- 목적지 도착 시 사용할 항구 이름 표식과 따뜻한 불빛 변화

| 캐릭터 | 선장 반신 1장 | 항구 도착과 정비 화면 공용, 추가 표정은 P1 |

원본은 PSD 또는 KRA로 보관하고 Unity 전달본은 투명 PNG로 납품한다. 파일명은 `범주_ID_상태` 형식으로 통일한다.

### 오디오

- T1은 높고 얇게, T2는 단단하게, T3는 낮고 무겁게 들리게 한다.
- 매 파괴마다 재화음을 따로 재생하지 않는다.
- 결정빙과 큰 연쇄에서만 짧은 강조음을 추가한다.
- 작업 종료음 뒤 정산 금액이 나타날 때 짧은 확인음을 1회 재생한다. 동전이 쏟아지는 소리는 사용하지 않는다.
- 일정이 부족하면 배경음악을 먼저 제외한다.

## 9. 밸런스 기준과 예상 플레이

### 표준·시연 데이터

| 항목 | 표준 | 시연 |
|---|---:|---:|
| 쇄빙 작업 | 60초 | 60초 |
| 항해 | 30초 | 10초 |
| 목적지 목표 | 120 / 600 / 2,400 | 40 / 120 / 300 |
| 정비 비용 | 표 기재값 | 표준 비용의 10%, 올림 |
| 전체 목표 | 30~60분 | 10분 이내 |
| 얼음 수 | 최대 20개 | 동일 |

표준과 시연은 같은 코드와 규칙을 사용하고 데이터 파일만 분리한다. 저장 파일도 서로 공유하지 않는다.

### 예상 작업 기록 v0

| 구간 | 60초 파괴 구성 | 기본 수입 | 목적지 진행 | 구매 예상 |
|---|---|---:|---:|---|
| 초반 | T1 20개 | 200 | +20 | 강화 장비 1개 |
| 중반 | T1 35개 + T2 15개 | 1,550 | +50 | 출력·보조·연쇄 중 1개 |
| 후반 | T1 18개 + T2 45개 + T3 27개 | 22,680 | +90 | 최종 정비 1개 수준 |

결정빙은 동일 단계의 기본 보상을 대체한다. 예를 들어 후반 기록에서 T3 하나가 결정빙이면 총수입은 2,100 증가한다. 한 번의 특수빙으로 최종 정비 여러 개를 건너뛰지 않도록 최종 비용을 15,000~20,000 구간에 둔다.

### 실제 조정 절차

1. 월요일 기본 반복부터 작업별 파괴 수와 단계별 비율을 기록한다.
2. 첫 구매가 첫 작업 뒤 가능한지 확인한다.
3. 초반 1~2회 작업마다 구매가 생기는지 확인한다.
4. T2와 T3 해금 직후 수입 효율이 체감되는지 확인한다.
5. 목표 플레이타임에서 세 목적지에 도달하는지 확인한다.
6. 화요일 통합본에서 1차 조정하고, 목요일 전체 플레이를 기준으로 최종 수치를 확정한다.

## 11. 개발 연결 규칙과 업무 분담

> 창 위치·피해 계산·연쇄 깊이·공용 이벤트·저장 및 재실행 경계는 [`07_Technical/implementation_lock.md`](../07_Technical/implementation_lock.md)를 따른다.

### 공용 데이터

- `IceDefinition`: 단계, HP, 기본 자금, 생성 조건
- `SpecialIceDefinition`: 확률, HP 배수, 자금 배수, 효과
- `MaintenanceDefinition`: ID, 계통, 단계, 효과, 비용, 선행 조건

- `GameState`: 현재 상태, 남은 시간, 자금, 목적지 진행, 구매 상태, 첫 파괴 여부, 쇄빙 시작 가능 여부

## 12. 검증 기준

### 시스템 시나리오

1. T1을 직접 파괴하면 자금 +10, 목적지 진행 +1, 보상 표시 1회가 발생한다.
2. T3 결정빙은 자금 +2,800, 목적지 진행 +1을 준다.
3. 보조 파쇄로 파괴해도 동일하게 처리한다.
4. 연쇄로 5개가 파괴되면 다섯 개의 자금을 합쳐 표시하고 목적지 진행은 +5가 된다.
5. 클릭과 연쇄가 동시에 들어가도 같은 얼음에서 두 번 지급하지 않는다.
6. 타이머 0 이후 도착한 피해에는 자금과 목적지 진행이 없다.
7. 작업 종료 시 자금이 추가로 증가하지 않고, 정산 화면의 획득액은 이번 작업의 실제 지급 합계와 같다.
8. 목적지 목표 달성 뒤에도 남은 60초 동안 자금은 계속 증가한다.
9. 구매 후 자금이 음수가 되지 않는다.
10. 저장 후 재실행하면 자금·목적지 진행·구매·첫 파괴 여부가 유지된다.

### UI 시나리오

1. 플레이 화면에 오른쪽 상태 패널이 없다.
2. 쇄빙 화면은 720×480 전체를 게임 영역으로 사용한다.
3. 쇄빙 화면에는 상단 왼쪽 정비 자금, 상단 중앙 타이머, 상단 오른쪽 설정만 고정 표시한다.
4. 쇄빙 화면에는 선박 정비·운항 현황·접기와 하단 고정 메뉴가 존재하지 않는다.
5. 첫 파괴는 재화 이름까지, 이후에는 금액만 표시한다.
6. 3개 이상 동시 파괴 시 보상 숫자를 합친다.
7. 화면 가장자리의 얼음을 파괴해도 보상 숫자가 X 24~696 / Y 60~344 밖으로 잘리지 않는다.
8. 보조 파쇄는 별도 숫자 패널 없이 장비의 대기·충전·발사 준비·발사 상태로 구분된다.
9. 축소 화면에서 정비 자금과 목적지 진행을 확인하며, 정산 종료 뒤 진행도가 약 0.4초 동안 증가한다.
10. 축소 화면의 정비 버튼은 선박 정비 화면, 운항 현황 버튼은 목적지 정보 화면을 중간 단계 없이 바로 연다.
11. 30초 항해 중 쇄빙 시작 버튼은 남은 시간을 표시하며 비활성이고, 30초 뒤 자동 확장 없이 `쇄빙 시작`으로 활성화된다.
12. 정비·운항 현황 확장 화면에서 탭을 전환하고, 접거나 준비 완료 뒤 쇄빙을 시작할 수 있다.
13. 연속 클릭으로 확장 화면이나 쇄빙 시작 이벤트가 두 번 생성되지 않는다.
14. 자동 확장과 포커스 강제 이동이 없다.
15. 작업 종료 뒤 자금 정산 화면이 매번 표시된다.
16. 정산 화면이 열린 뒤 1.2초 전에는 버튼·클릭·Enter·Space 입력으로 넘길 수 없다.
17. 1.2초 이후에는 입력으로 항해를 시작할 수 있고, 입력이 없으면 화면이 열린 시점부터 4초 뒤 자동으로 항해한다.
18. 30초 항해 타이머는 정산 화면이 닫힌 뒤 시작한다.
19. 정산 화면의 자금·파괴 수·목적지 진행 증가는 `SettlementSummary`와 같고 화면에서 다시 지급하지 않는다.
20. 목적지 도달 시 배지와 항구 이름을 표시하고 정산 뒤 1~2초 도착 연출을 재생한다.
21. 목적지 도달은 T2·T3를 해금하지 않으며, 해당 얼음은 정비 구매로만 해금된다.
22. 설정 화면이 열려 있는 동안 쇄빙·항해 타이머와 얼음 생성·피해 진행이 멈추고, 닫으면 멈춘 시점부터 재개된다.
23. 최초 실행 시 항해 대기 없이 바로 쇄빙을 시작할 수 있다.

### 완료 정의

- 7월 19일 일요일: T1 파괴 결과가 자금과 HUD까지 연결된다.
- 7월 20일 월요일: 항해부터 정산까지 한 사이클을 반복할 수 있다.
- 7월 21일 화요일: 정비 13개, 특수빙, 보조 파쇄, 연쇄가 1차 통합된다.
- 7월 22일 수요일: 저장 후 재실행과 처음부터 목적지 도달까지 가능한 Windows 빌드를 만든다.
- 7월 23일 목요일: 전체 플레이를 기준으로 밸런스·연출·아트·오디오를 보완한다.
- 7월 24일 금요일: 다른 PC 실행, 해상도, 저장, 정산 수치, 제출 파일을 최종 확인한다.
- 수요일 이후에는 새로운 핵심 시스템을 추가하지 않는다.

- v2.6: 얼음 크기(68~84px)·원형 클릭 판정(반지름 44px)·최소 생성 간격(88px)과 해금 상태별 생성 확률표를 확정했다. 최초 실행 즉시 시작, 앱 종료 중 절대 시각 기반 항해 타이머 진행, 3항목 설정 화면과 설정 중 일시정지, 별도 타이틀 화면 없음을 잠갔다. 개발자의 경험 차이를 반영해 민준은 진행·경제·정비·저장·통합, 시연은 얼음 파쇄 게임플레이, 정환은 HUD·피드백·정산 표시·QA를 담당하도록 재배치했다. 7월 19일 일요일부터 22일 수요일까지 기능을 완성하고, 23일 목요일과 24일 금요일 대면 해커톤에서 품질 보완과 제출을 진행하도록 WBS를 변경했다.
- v2.5: 선박 원화와 계통별 탭을 제거하고 13개 강화를 한 화면에 표시하는 고정형 연결 트리로 확정했다. 노드 상태·선행 조건·상세 패널·아트 및 개발 담당 범위를 함께 잠갔다.
