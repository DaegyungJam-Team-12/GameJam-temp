# 게임 상태 흐름

> 기준 문서: `new/ICEBREAKER_SPEC_v2.md` v2.6  
> 60초 쇄빙 작업은 성공·실패 판정이 아니라 고정 길이의 성과 측정 구간이다.

## 전체 반복 구조

```text
항해 및 정비 대기
→ 다음 쇄빙까지 30초
→ 축소 화면의 쇄빙 시작 버튼 활성화
→ 사용자가 쇄빙 시작
→ 3·2·1 카운트다운
→ 쇄빙 작업 60초
→ 작업 종료 1.2초
→ 자금 정산 화면(최소 1.2초·최대 4초)
→ 축소 화면 복귀·다음 항해 30초
```

## 상태별 규칙

| # | 상태 | 내용 |
|---|---|---|
| 1 | 항해 중 | 480×56 축소 게임 바 상태. 정비 자금, 현재 목적지 진행, 정비, 운항 현황, 쇄빙 시작, 설정을 표시한다. 쇄빙 시작 버튼은 `다음 쇄빙 00:24`처럼 남은 시간을 표시하며 비활성이다. 정비·운항 현황 화면을 계속 이용할 수 있다. |
| 2 | 쇄빙 준비 완료 | 30초가 끝나면 자동으로 창을 펼치거나 포커스를 가져오지 않고 쇄빙 시작 버튼만 활성화한다. 시작 버튼을 한 번만 짧게 점등한다. 사용자가 누르기 전까지 정비·운항 현황·설정을 계속 이용할 수 있다. |
| 3 | 카운트다운 | 사용자가 축소 화면 또는 정비·운항 현황 화면의 `쇄빙 시작` 버튼으로 직접 시작한다. 3·2·1 표시, 입력 불가. 정비·운항 현황 화면은 닫힌다. |
| 4 | 쇄빙 작업 | 정확히 60초. 클릭·누르고 있기로 얼음을 공격한다. 720×480 전체가 게임 영역이며 상단에 정비 자금·타이머·설정만 띄운다. 선박 정비, 운항 현황, 접기 메뉴는 화면에 표시하지 않는다. 목적지 목표를 도중에 달성해도 남은 시간 동안 정비 자금을 계속 획득한다. |
| 5 | 작업 종료 | 타이머가 0이 되면 입력, 얼음 생성, 진행 중 피해를 즉시 차단한다. 0.2초간 정지한 뒤 남은 1.0초 동안 배경 얼음판의 균열·파괴와 정산 화면 전환을 진행한다. |
| 6 | 자금 정산 | 매 쇄빙 작업 뒤 반드시 표시한다. `획득 정비 자금`, `파괴한 얼음 수`, `목적지 진행 증가량`을 표시한다. 최소 1.2초 유지 후 버튼·화면 클릭·Enter·Space로 넘길 수 있고, 입력이 없으면 열린 시점부터 4초 뒤 자동으로 항해로 전환한다. |
| 7 | 축소 화면 복귀 | 30초 항해 타이머는 정산 화면이 닫힌 뒤 시작한다. 목적지 진행도가 이전 값에서 새 값까지 약 0.4초 동안 오른다. |

## 첫 실행과 반복

- 최초 실행은 항해 대기 없이 쇄빙 준비 완료 상태에서 시작한다.
- 앱 종료 중에도 다음 쇄빙 가능 시각은 절대 시각으로 진행된다. 재실행 시 시각이 지났으면 바로 준비 완료 상태다. 앱 종료 중 자금·목적지 진행은 변하지 않는다.
- 설정 화면이 열려 있는 동안 게임을 일시정지한다. 쇄빙 타이머·항해 타이머·얼음 생성·진행 중 피해를 멈추고, 닫으면 멈춘 시점부터 재개한다.
- 자금과 목적지 진행은 얼음 파괴 순간 즉시 반영된다. 정산 화면은 요약만 하고 추가 지급하지 않는다.
- 항해 중에는 자동 파쇄·자동 재화 획득·자동 목적지 진행이 없다.
- 연속 클릭으로 쇄빙 시작 이벤트가 두 번 발생하지 않게 한다.

## 다음 쇄빙까지 시간

| 빌드 | 시간 |
|---|---|
| 최초 실행 | 즉시 (대기 없음) |
| 표준 | **30초** |
| 시연 | 10초 |

표준과 시연은 같은 코드와 규칙을 사용하고 데이터 파일만 분리한다.

## 목적지 도착 분기

1. 얼음 파괴 시마다 현재 목적지 진행도가 목표 미만이면 +1 증가한다.
2. 목표에 도달하면 이후 파괴는 자금만 지급한다. 초과분은 다음 목적지로 넘기지 않는다.
3. 목적지 도착 여부는 작업 종료 때 확인한다.
4. 도착 시 정산 패널 상단에 `목적지 도달` 배지와 항구 이름을 추가하고, 정산 화면이 닫힌 뒤 1~2초 도착 연출을 재생한다.
5. 북쪽 기지 도착이 P0 완료 조건이다. 별도 엔딩은 제출 후보가 안정된 뒤에만 검토한다.

## 시간 경계 규칙

- 얼음 HP가 0이 된 시점이 타이머 0 이전일 때만 파괴 이벤트를 인정한다.
- 타이머 0 이후 도착한 투사체와 연쇄 피해는 취소한다.

## 자금 정산 표시

- 획득 정비 자금 (이번 작업에서 승인된 `IceDestroyed` 이벤트의 합계)
- 파괴한 얼음 수
- 목적지 진행 증가량 (목표 초과분 제외한 실제 증가량)
- 목적지 도달 시: `목적지 도달` 배지와 항구 이름

`정비 자금 반영 완료`를 함께 표시해 종료 시 다시 지급되는 금액이 아님을 알린다.
