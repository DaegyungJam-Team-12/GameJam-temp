## 정비 트리 연결 관계

Step ID는 `<정비 ID>-L<목표 단계>` 형식을 사용하며 Layout과 런타임 View를 연결하는 키다.

```text
C01-L1
├─ C02-L1 → C02-L2 → C02-L3
├─ C03-L1 → C04-L1
├─ D01-L1 → D01-L2 → D01-L3
│  ├─ D01-L1 → S01-L1 → S02-L1 → S02-L2
│  │                         └→ S03-L1
│  └─ D01-L2 → D03-L1
│             └→ H01-L1 → H01-L2 → H01-L3 → H03-L1
│                        └→ H02-L1 → H02-L2
└─ D02-L1 → D02-L2 → D02-L3
            └→ D04-L1 → D04-L2 → D04-L3
```

논리 정비의 첫 구매 조건은 C02·C03·D01·D02는 C01, C04는 C03, D03·H01은 D01 2단계, D04는 D02 1단계, S01은 D01 1단계, S02는 S01, S03은 S01과 S02 1단계, H02는 H01 1단계, H03은 H01 3단계다. 다단계 정비의 다음 단계는 같은 정비의 직전 Step 구매를 자동 선행으로 사용한다.

### 기존 저장 호환

- 현재 레벨이 1 이상인 논리 정비는 새 첫 구매 조건을 충족한 것으로 본다.
- 이미 구매한 논리 정비의 다음 단계에는 새 첫 구매 조건을 다시 강제하지 않는다.
- 구매한 레벨을 낮추거나 환불하지 않으며 논리 정비 ID도 변경하지 않는다.
- 현재 레벨이 0인 논리 정비의 첫 구매에만 새 선행 조건을 검사한다.

## 화면 표시 규칙

- 정비 화면은 14개 논리 정비를 26개 시각 구매 Step으로 펼친 약 1600×900 공간형 트리다.
- `공통 장비 / 직접 파쇄 / 보조 파쇄 / 연쇄 파괴`는 별도 탭이 아니라 같은 공간 안의 가지 이름으로 표시한다.
- 960×540 화면에서 빈 공간·노드 드래그 팬, WASD 팬, 휠 확대·축소(0.8~1.25)를 지원한다. 별도 스크롤바는 두지 않는다.
- 연결선은 선행 조건을 표현하며 Layout에 저장된 직교 꺾임점을 사용해 노드 뒤에 배치한다.
- 상태는 `구매됨 / 구매 가능 / 잠김`의 구매 상태와 `표시 / 한 단계 미리보기 / 숨김`의 공개 상태를 별도로 표현한다.
- 구매 가능하지만 자금이 부족한 Step은 외곽선을 유지하고 가격 경고를 함께 표시한다. 색만으로 상태를 구분하지 않는다.
- 초기에는 `C01-L1`이 구매 가능·표시되고 직접 자식만 잠김·미리보기로 보인다. 구매됨 또는 구매 가능한 Step의 직접 자식까지만 미리보기로 공개한다.
- 노드를 선택하면 일정한 읽기 크기의 Overlay 카드에 이름, 현재/최대 단계, 해당 Step 효과, 현재값→구매 후 값, 가격, 잠금 조건을 표시한다.
- 구매 완료 시 해당 Step과 다음에 열리는 연결선이 점등된다. 숨김 Step은 표시·Raycast·직접 구매 요청 대상이 아니다.
- 구매 화면은 가격·선행·구매 가능 여부를 다시 계산하지 않고 Core의 Step ViewData를 그대로 표시한다.

### 정비 트리 UI 전달 데이터

- Step ID, 논리 정비 ID, 이름과 계통
- 목표 단계와 최대 단계
- `구매됨 / 구매 가능 / 잠김` 구매 상태
- `표시 / 한 단계 미리보기 / 숨김` 공개 상태
- 해당 Step 효과 문구와 비용
- 자금 충족·구매 가능 여부
- 충족하지 못한 선행 정비 ID

## 구매·저장·적용 시점

```text
구매 성공
→ 현재 자금과 논리 정비 레벨 즉시 변경
→ funds와 maintenanceLevels를 한 번의 Flush로 저장
→ UI 전체 Step을 즉시 다시 조회해 갱신
→ 다음 쇄빙 시작 요청 때 최신 정비 레벨로 CombatConfig 생성
→ 카운트다운 전에 IceFieldView에 주입
→ 쇄빙 종료까지 해당 설정 불변
```

- 정비 화면은 `GamePhase`가 아닌 별도 관리 화면 상태다. Traveling/Ready를 유지하며 타이머도 계속 진행한다.
- Countdown부터 Arrival까지는 정비 구매와 정비 화면 진입을 허용하지 않는다.
- 구매 결과가 현재 진행 중인 쇄빙의 피해·틱·범위·보상 효율을 변경하지 않는다.

## 효과 계산 규칙

구현자가 계통별 효과를 다르게 해석하지 않도록 아래를 고정한다.

```text
직접 피해 = D01 단계별 최종값 1 / 3 / 5 / 7
치명타 = 5% 확률 · 피해 ×3
정비 자금 증가 = 내림(단계 기본 지급액 × 특수빙 배수 × (1 + C02 단계 × 0.10))
```

- 같은 종류 보정(C02, H02)은 합연산이며, D01은 단계별 최종값을 사용한다.
- 결정빙 배수와 정비 효율은 서로 곱한다.
- D03 전이 피해와 보조탄에는 자체 치명타가 발생하지 않는다.
- 현재 자금은 정수로 저장하며 0보다 작아질 수 없다. 구매 후 자금이 음수가 되지 않는다.

## 설계 규칙

- 보조 파쇄는 직접 조작의 보상이며 방치만으로 쇄빙 작업을 완료하지 않는다.
- 완성 그림: 강한 직접 타격 → 보조탄 추가 타격 → 파편 비산 → 특수빙 연쇄 → 빙판 붕괴가 연달아 발생한다.
- 비용과 효과 수치는 `03_Data/balance_tables.md`를 단일 출처로 사용한다.
