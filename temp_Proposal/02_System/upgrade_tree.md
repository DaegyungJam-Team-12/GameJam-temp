## 화면 표시 규칙

- 정비 화면은 13개 강화가 한 화면에 모두 보이는 **고정형 연결 트리**로 구성한다.
- `공통 장비 / 직접 파쇄 / 보조 파쇄 / 연쇄 파괴`는 별도 탭이 아니라 한 보드 안의 가지 이름으로만 표시한다.
- 계통별 탭, 스크롤, 확대·축소, 드래그 이동은 사용하지 않는다.
- 연결선은 선행 조건을 표현하며 아이콘과 이름을 통과하지 않게 노드 뒤에 배치한다.
- 노드에는 아이콘과 현재 단계만 표시한다. 노드를 선택하면 하단 상세 패널에 이름, 현재 단계, 현재 효과, 다음 효과, 다음 비용, 구매 버튼을 표시한다.
- 상태는 `보유 / 구매 가능 / 잠김` 세 가지로 제한하며, 선택한 노드는 현재 상태 위에 이중 청색 외곽선을 추가한다.
- 구매 완료 시 해당 노드와 다음에 열리는 연결선이 점등된다.
- 선행 조건을 만족하지 않은 노드는 자물쇠와 필요한 선행 노드를 표시한다.
- 구매 화면은 효과나 비용을 다시 계산하지 않고 `MaintenanceDefinition`과 노드 상태 데이터를 그대로 표시한다.

- 현재 단계와 최대 단계

## 효과 계산 규칙

구현자가 계통별 효과를 다르게 해석하지 않도록 아래를 고정한다.

```text
직접 피해 = 기본 클릭 피해 × 1.6^(D01 단계)
치명타 = 5% 확률 · 피해 ×3
정비 자금 증가 = 내림(단계 기본 지급액 × 특수빙 배수 × (1 + C02 단계 × 0.10))
```

- 같은 종류 보정(C02, H02)은 합연산, D01 단계끼리는 곱연산이다.
- 결정빙 배수와 정비 효율은 서로 곱한다.
- D03 전이 피해와 보조탄에는 자체 치명타가 발생하지 않는다.
- 현재 자금은 정수로 저장하며 0보다 작아질 수 없다. 구매 후 자금이 음수가 되지 않는다.

## 설계 규칙

- 보조 파쇄는 직접 조작의 보상이며 방치만으로 쇄빙 작업을 완료하지 않는다.
- 완성 그림: 강한 직접 타격 → 보조탄 추가 타격 → 파편 비산 → 특수빙 연쇄 → 빙판 붕괴가 연달아 발생한다.
- 비용과 효과 수치는 `03_Data/balance_tables.md`를 단일 출처로 사용한다.
