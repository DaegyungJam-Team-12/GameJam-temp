# 얼음 생성 규칙과 목적지 진행

> 기준 문서: `new/ICEBREAKER_SPEC_v2.md` v2.6  
> 한 번의 60초 플레이는 항해 전체가 아니라 한 차례의 쇄빙 작업이다.

## 쇄빙 화면

- 960×540 전체를 게임 영역으로 사용하고, 상단에는 정비 자금·타이머·설정만 띄운다.
- 화면 동시 존재 최대 20개 얼음. 보이는 격자와 행·열 슬롯은 사용하지 않는다.
- 얼음은 서로 겹치지 않는 자유 위치에 생성한다.
- 얼음 스프라이트는 96~112px를 초기 표시값으로 사용하고, 클릭 판정은 얼음 중심 기준 반지름 56px 원형이다.
- 생성 시 얼음 중심 사이 최소 거리 120px를 우선 적용한다. 30회 시도 안에 자리를 찾지 못하면 104px까지 완화해 배치한다.
- 선박 뱃머리와 보조 파쇄 장비는 게임 영역 하단에 포함하며, 장식이 아니라 실제 시스템 상태를 전달하는 게임 오브젝트다.

### 얼음과 선박 안전 영역

- 정비 자금 보호 영역: X 0~252 / Y 0~64
- 타이머 보호 영역: X 384~576 / Y 0~64
- 설정 보호 영역: X 888~960 / Y 0~64
- 선박·보조 파쇄 장비 영역: X 280~680 / Y 405~540
- 얼음 배치 외곽 여백: 20px
- 최대 20개의 얼음은 보호 영역과 외곽 여백을 침범하지 않게 생성한다.
- 보상 숫자는 파괴 위치를 기준으로 하되 X 28~932 / Y 64~389 안으로 보정한다.

## 재생성

1. 파괴 후 0.12초 공백
2. 0.18초 생성 연출을 거쳐 다시 등장
3. 연쇄로 여러 개가 파괴되면 0.03초 간격으로 순서대로 생성
4. 다시 생성된 얼음은 0.25초 동안 연쇄 피해를 받지 않음

- 부서진 얼음은 작업 시간 안에 다시 생성된다.
- 새 얼음은 파괴된 위치에 고정 리필하지 않고, 현재 얼음들과 겹치지 않는 새 자유 위치를 다시 선택한다.

## 얼음 단계 등장

- 첫 쇄빙 작업은 전부 T1 백빙이다.
- T2 청빙은 선박 정비 `청빙 대응` 구매 후, T3 심빙은 `심빙 대응` 구매 후 등장한다.
- 목적지는 T2·T3 얼음을 해금하지 않는다.
- 새 얼음이 등장해도 이전 단계는 사라지지 않는다. 강해진 플레이어가 낮은 단계를 빠르게 정리하는 장면을 계속 만든다.

### 얼음 단계 생성 확률

| 해금 상태 | 생성 확률 |
|---|---|
| 기본 | T1 100% |
| 청빙 대응 구매 후 | T1 60% · T2 40% |
| 심빙 대응 구매 후 | T1 20% · T2 45% · T3 35% |

- 재생성마다 독립적으로 판정한다.
- 특수빙 여부는 단계 결정 후 별도로 판정한다.
- 이 표는 v0 초기값이며 화요일 통합본에서 1차 조정한다.

## 목적지 진행

- 얼음 하나를 파괴할 때마다 현재 목적지 진행도가 `+1` 증가한다.
- 얼음 단계, 특수빙, 파괴 원인(직접·보조·연쇄)은 진행 증가량에 영향을 주지 않는다.
- 현재 목적지 목표에 도달하면 이후 파괴는 자금만 지급한다.
- 초과분은 다음 목적지로 넘기지 않는다.
- 목적지 도착 여부는 작업 종료 때 확인한다.

| 구간 | 목적지 | 화물 | 표준 목표 | 시연 목표 |
|---:|---|---|---:|---:|
| 1 | 섬마을 | 식료품·우편 | 120 | 40 |
| 2 | 등대항 | 발전기 연료·의약품 | 600 | 120 |
| 3 | 북쪽 기지 | 기계 부품·우편 | 2,400 | 300 |

- 수치의 단일 출처는 `03_Data/balance_tables.md`다.
- 쇄빙 중 목적지 진행 수치를 표시하지 않는다. 축소 화면과 운항 현황 화면에서 확인한다.
- P0 목적지는 정해진 순서로 열며 경로 선택은 구현하지 않는다.
- 북쪽 기지 도착이 P0 완료 조건이다.

## 연쇄 파괴 처리

- 특수빙이 특수빙을 깨면 그 효과도 발동한다. 연쇄는 최대 3단계다.
- 새로 생성된 얼음은 진행 중인 연쇄에 맞지 않는다. 생성 직후 0.25초 면역을 적용한다.
- 범위 효과의 대상 판정은 얼음 중심점 사이 거리로 계산한다.
- 같은 얼음에 직접·보조·연쇄 피해가 동시에 들어와도 파괴 처리는 한 번만 발생한다.

## 작업 종료 전환

- 타이머가 0이 되면 입력, 얼음 생성, 진행 중 피해를 즉시 차단한다.
- 0.2초간 정지한 뒤 남은 1.0초 동안 배경 얼음판의 균열·파괴와 정산 화면 전환을 진행한다.
- 배경 얼음판은 기본·균열·파괴 3상태 에셋으로 표현하며, 별도 HP나 충돌 판정을 가진 오브젝트가 아니다.
