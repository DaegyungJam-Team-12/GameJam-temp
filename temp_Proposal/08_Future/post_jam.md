# P1 — 게임잼 이후 로드맵

> 기준 문서: `new/ICEBREAKER_SPEC_v2.md` v2.6  
> P1은 P0를 더 크게 만드는 단계다. P0 완주 데이터와 플레이테스트 없이 자동화·온라인·경제 시스템을 먼저 추가하지 않는다.

## 1. P1 진입 조건

- P0 신규 플레이어 완주율과 평균 완주 시간이 측정됨
- 파쇄 손맛, 목적지 진행, 정비 선택의 세 축이 플레이테스트에서 이해됨
- Windows 창 제어와 저장의 차단 버그가 없음
- 개발을 이어갈 팀 일정과 배포 플랫폼이 확정됨

## 2. P1 항목

- T4·T5와 추가 특수빙
- 추가 목적지와 분기 경로
- 의뢰 선택과 화물 보정
- 통계와 기록 비교 확장
- 영어를 포함한 추가 언어
- macOS·Steam 출시 준비

의뢰 선택은 선택의 의미를 만드는 시스템이며, 자유 인벤토리나 물류 시뮬레이션으로 확장하지 않는다.

## 3. P0 일정 여유 시 항목 (P1 아님)

아래는 게임잼 안에서 일정이 남을 때만 추가하는 항목이며, 미구현이어도 P0는 완결된다.

- 목적지별 고유 항구 연출
- 영어
- 선택 엔딩
- 대표 이미지와 추가 배경음악
- 장식용 선원과 장비 외형 변화
- 최초 실행 1~2초 로고 스플래시

## 4. P1에서도 기본 제외

- 자동 공격과 앱 종료 중 수익
- 거래 경제와 아이템 현금화
- 선체 내구도와 실패 판정
- 장문 대화와 물류 시뮬레이션
- 동전·부품 드롭과 수동 획득
- 온라인 멀티플레이

자동화는 직접 파쇄의 핵심을 약하게 하므로 기본 로드맵에 넣지 않는다. 플레이테스트에서 반복 피로가 명확히 확인되면, 자동 공격보다 입력 보조·홀드 편의부터 검토한다.

## 5. 검증해야 할 질문

1. 60초 작업을 여러 번 반복해도 강화 전후 차이가 충분히 큰가?
2. 목적지와 화물이 설명문 없이 반복 플레이의 목적이 되는가?
3. 세 목적지를 완주한 유저가 의뢰 선택을 원하는가, 더 많은 얼음 단계를 원하는가?
4. 30초 항해가 데스크톱 컴패니언 리듬을 만드는가, 단순한 중단으로 느껴지는가?
5. 미형 선장 원화가 실제 플레이 화면과 브랜드를 연결하는가?

이 답을 얻기 전에는 P1 세부 규모를 잠그지 않는다.
