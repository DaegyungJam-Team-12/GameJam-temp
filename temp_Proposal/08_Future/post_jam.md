# P1 — 게임잼 이후 로드맵

> P1은 P0를 더 크게 만드는 단계다. P0 완주 데이터와 플레이테스트 없이 자동화·온라인·경제 시스템을 먼저 추가하지 않는다.

## 1. P1 진입 조건

- P0 신규 플레이어 완주율과 평균 완주 시간이 측정됨
- 파쇄 손맛, 항로 목적, 정비 선택의 세 축이 플레이테스트에서 이해됨
- Windows 창 제어와 저장의 차단 버그가 없음
- 개발을 이어갈 팀 일정과 배포 플랫폼이 확정됨

## 2. P1 우선순위

### P1-A — 콘텐츠 확장

- T4 자빙·T5 극광빙과 더 먼 북쪽 항로
- 흑빙·서리빙과 대응 정비
- 목적지·화물·거리 배경 추가
- 3개 정비 계통의 추가 노드와 플레이 통계
- 정비 화면의 장비 외형 변화와 선장 표정 소수
- 완주 뒤 계속 플레이 또는 본편 엔딩

### P1-B — 운항 선택

- 여러 계약 중 목적지·거리·보수를 보고 하나 선택
- 화물별 단순 운항 조건 또는 보정
- 분기 항로와 재방문
- 계약은 선택의 의미를 만드는 시스템이며, 자유 인벤토리나 물류 시뮬레이션으로 확장하지 않는다.

### P1-C — 제품화

- macOS 검증
- 8~11개 언어 현지화
- Steam 저장·도전 과제·배포 페이지
- 접근성·키 재설정·추가 해상도와 DPI QA
- 본편용 BGM과 확장 사운드
- 플레이 세션에 맞춘 8~15분 항해 대기 재튜닝

## 3. P1에서도 기본 제외

- 상시 자동 공격과 앱 종료 중 재화 획득
- 현금화·거래·시장 경제
- 선박 전투, 선체 HP, 게임 오버
- 장문의 대화·퀘스트 로그·항구별 컷신
- 복잡한 화물 무게·적재 퍼즐
- 온라인 멀티플레이

자동화는 직접 파쇄의 핵심을 약하게 하므로 기본 로드맵에 넣지 않는다. 플레이테스트에서 반복 피로가 명확히 확인되면, 자동 공격보다 입력 보조·홀드 편의·짧은 구간 스킵부터 검토한다.

## 4. 검증해야 할 질문

1. 60초 작업을 여러 번 반복해도 강화 전후 차이가 충분히 큰가?
2. 항로와 화물이 설명문 없이 반복 플레이의 목적이 되는가?
3. 고정 항로를 완주한 유저가 계약 선택을 원하는가, 더 많은 빙질을 원하는가?
4. 항해 대기가 데스크톱 컴패니언 리듬을 만드는가, 단순한 중단으로 느껴지는가?
5. 미형 선장 원화가 실제 플레이 화면과 브랜드를 연결하는가?

이 답을 얻기 전에는 P1-B·P1-C의 세부 규모를 잠그지 않는다.

## 5. 단계별 출시 후보

| 단계 | 내용 | 판단 기준 |
|---|---|---|
| P1 프로토타입 | T4 1종, 신규 목적지 1곳, 계약 선택 2개 | 기존 코어를 해치지 않고 선택이 생김 |
| 비공개 테스트 | 1~2시간 콘텐츠, 저장·대기·완주 반복 | 완주·재방문·이탈 지점 확인 |
| 공개 데모 | 첫 항로군과 대표 정비 계통 3개 | 30분 안에 게임 정체성 전달 |
| 제품판 | 확정된 항로·빙질·현지화·플랫폼 기능 | 품질과 일정 재산정 후 결정 |

가격, 전체 플레이타임, 출시일은 P0 결과와 시장 검증 뒤 별도로 결정한다.
