# ICEBREAKER — 프로젝트 개요

> 얼어붙은 바다의 마지막 쇄빙 보급선  
> 게임잼 「납량(서늘함을 들인다)」 출품작 · v3.2 · 2026.07

## 한 줄 소개

**얼어붙은 바다를 항해하는 마지막 쇄빙선을 조작해 유빙을 부수고, 고립된 섬에 보급품을 전달하는 60초 파쇄 증분 게임.**

## 유저 카피

**얼어붙은 바다를 뚫고, 끊어진 항로를 잇는다.**

## 플레이어가 하는 일

1. 최대 20개의 얼음을 클릭·홀드로 부순다.
2. 60초 동안 번 운항 보수와 전진 점수를 정산한다.
3. 정비 자금으로 쇄빙 장비를 개조한다.
4. 다음 유빙 구간을 돌파하며 항구에 가까워진다.
5. 식료품·연료·의약품·기계 부품·우편을 세 목적지에 전달한다.

## 핵심 스펙

| 항목 | P0 확정 |
|---|---|
| 장르 | 파쇄·운항형 증분 × 60초 액션 × taskbar 컴패니언 |
| 코어 판 | 60초 · 자유 위치 유빙 구간 · 최대 20개 얼음 |
| 성장 | T1~T3 빙질 대응 + 직접·보조·연쇄 3개 정비 계통 |
| 재화 | 정비 자금 1종 |
| 장기 목표 | 고정 항로 3개를 열어 북쪽 기지에 도착 |
| 판 사이 | 다음 유빙 구간까지 표준 30초 항해, 자동 파쇄·방치 수익 없음 |
| 창 | 480×56 도킹 / 720×480 확장 |
| 플랫폼 | Windows 10/11 |
| 엔진 | Unity 6 + C# + UniWindowController |
| 언어 | 한국어·영어 |
| 목표 완주 | 신규 저장 기준 30~60분 · D5~D10 플레이테스트로 튜닝 |

## 성장의 정의

숫자가 커지는 것보다 **한 번의 입력으로 화면에서 함께 깨지는 얼음이 늘어나는 것**을 먼저 보여준다.

```text
한 얼음을 더 빠르게 파쇄
→ 보조 장비가 다른 얼음을 추가 타격
→ 충격과 균열이 주변으로 전파
→ 더 단단한 빙질을 열고 이전 얼음을 화면째 정리
```

## 이야기 전달 원칙

- 프롤로그·대화·컷신으로 세계를 설명하지 않는다.
- 뱃머리, 갑판의 화물, 항로 지도, 가까워지는 항구, 도착 후 켜지는 불빛으로 목적을 전달한다.
- 화물은 P0에서 시각 정보이며 인벤토리나 능력치가 아니다.
- 항구 도착은 배경·화물·지도 상태를 바꾸는 1~2초 공통 시스템 연출이다.

## P0 / P1 경계

### P0 — 게임잼에 반드시 완결

- T1~T3, 결정빙·균열빙
- 고정 항로 3개와 북쪽 기지 공통 도착 처리
- 구매 13개, 로컬 저장, 한국어·영어
- 캐주얼 핸드페인팅 게임 화면과 선장 반신 원화 1장

전용 엔딩 화면·추가 엔딩 연출은 P0 필수가 아니다. RC가 안정되고 시간이 남을 때만 스프린트 최후위 옵션으로 제작한다.

### P1 — 플레이테스트 뒤 검토

- T4·T5, 흑빙·서리빙, 추가 목적지
- 계약 선택·분기 항로·단순 화물 조건
- macOS·다국어·Steam 제품화

자유 항로, 화물 효과, 자동 파쇄는 P0에 포함하지 않는다.

## 팀 구성과 14일 목표

- 개발자 1: 얼음·파쇄 코어
- 개발자 2: 상태·항로·정비·저장·데이터
- 개발자 3: 창·UI·메인 씬·통합·빌드
- 아티스트 1: 게임 2D 에셋·선장 원화

D5 코어 수직 슬라이스, D8 메타 루프, D11 콘텐츠 동결, D13 RC, D14 제출을 기준으로 한다.
