# P0 (게임잼) 범위 잠금

> **상태: LOCKED · v3.2 · 2026-07-18**  
> 목표는 새 기능을 많이 보여주는 것이 아니라, **60초 파쇄 → 정산 → 정비 → 다음 유빙 구간 → 항구 도착**이 하나의 완결된 게임으로 이어지는 것이다.

## 1. P0 성공 기준

- 표준 빌드에서 신규 플레이어가 목표 30~60분 안에 북쪽 기지에 도착하고 세 항로의 완료 상태를 확인한다. 이 시간은 D5~D10 플레이테스트로 튜닝한다.
- 첫 5분 안에 `직접 파쇄 → 운항 보수 → 선박 정비 → 더 큰 파괴`의 성장 차이를 이해한다.
- 대사나 프롤로그 없이도 `화물을 실은 배가 얼음을 깨고 항구로 간다`는 상황이 보인다.
- Windows 10/11의 일반 해상도와 100%·125%·150% DPI에서 도킹·확장 전환이 플레이를 막지 않는다.
- 20개 얼음과 연쇄 파괴가 동시에 발생해도 목표 60fps를 유지한다.
- 저장 후 재실행했을 때 정비·항로·대기 시간이 이어진다.

## 2. 반드시 구현 — P0

### 코어 파쇄

- 60초 고정 쇄빙 작업, 7단계 상태 흐름, 마지막 5초 경고 이벤트
- 520×376px 자유 위치 유빙 구간, 최대 20개 얼음
- 클릭·홀드 피해, 균열 단계, 파괴·재스폰, 타격 피드백
- T1 백빙·T2 청빙·T3 심빙과 티어 혼합·신규 티어 보장
- 결정빙·균열빙, 최대 3단 연쇄, 최대 연쇄 기록
- 전진 점수와 작업별 운항 보수 정산

### 항로와 완주

- 선택 없는 고정 항로 3개

| 순서 | 목적지 | 화물 | 항로 목표 |
|---:|---|---|---:|
| 1 | 섬마을 | 식료품·우편 | 120 |
| 2 | 등대항 | 발전기 연료·의약품 | 600 |
| 3 | 북쪽 기지 | 기계 부품·우편 | 2,400 |

- 파쇄 전진 점수를 현재 항로 진행에 1:1 반영
- 항로 완료 시 공통 1~2초 도착 연출 후 다음 항로 자동 개방
- 북쪽 항로도 목표값 도달로 완료하고 공통 항구 도착 연출을 재사용
- T3 심빙 첫 파괴는 성장 이정표로 기록하며 항로 완료 조건으로 사용하지 않음
- 화물은 목적을 보여주는 아이콘이며 인벤토리·수량·효과가 없다.

### 성장과 대기

- 단일 재화: 운항 중 획득하는 `운항 보수`를 정산 후 `정비 자금`으로 보유
- 구매 항목 13개: 기본 파쇄 장비 1 + 공통 진행 3 + 정비 계통 9
- 직접 파쇄·보조 파쇄·연쇄 파괴 3개 정비 계통
- 보조 파쇄는 클릭·파괴·연쇄에만 반응하며 상시 자동 공격 없음
- 다음 유빙 구간까지 표준 30초 항해, 앱 종료 중에는 대기 시간만 진행
- 항해 중 정비·기록·항로 확인 가능, 다음 작업은 유저가 직접 시작

### 창·UI·저장

- 480×56 도킹 스트립과 720×480 확장 화면
- 도킹·확장·포커스 복귀·DPI 대응, 실패 시 하단 스냅 일반 창 폴백
- 준비·플레이·정산 상태에 따라 바뀌는 우측 패널
- 목적지·화물·항로 진행·현재 빙질·보수·대기 시간 표시
- 선박 측면 정비 화면과 장비 영역 4곳의 선택 하이라이트
- 로컬 JSON 저장: 정비, 항로, 대기 시각, 3개 항로 완료, 설정
- 한국어·영어, 소리 기본 꺼짐, 볼륨·창 고정·화면 흔들림 설정

### 아트·오디오 최소선

- `05_Art/asset_list.md`의 P0-A·P0-B 에셋
- T1~T3 얼음, 균열·파괴·특수빙 오버레이, 선박 전면·측면, 거리 배경, 목적지 3곳, 화물 5종, 지도·UI
- 선장 반신 원화 1장. 표정 세트와 캐릭터 애니메이션 없음
- `06_Audio/audio.md`의 P0-A 사운드. P0-B는 통합 상황에 따라 적용
- 파쇄·기관·항구 도착 SFX 중심. 음성 대사 없음

## 3. 별도 시연 프로필

시연 프로필은 발표 시간을 위한 **데이터 프리셋**이며, 유저에게 보이는 난이도 옵션이나 별도 게임 모드가 아니다.

- 다음 유빙 구간까지 10초
- 항로 목표 40 / 120 / 300
- 정비 비용 초기값은 `max(1, ceil(표준값 × 0.1))`
- 실행 인자 또는 시연용 빌드 설정으로만 선택
- 표준·시연 프로필은 같은 코드와 저장 구조를 사용하되 저장 파일은 분리

## 3-1. 스프린트 최후위 선택 항목

아래 항목은 P0 필수가 아니다. D12의 표준 RC가 안정되고 차단·주요 버그가 없을 때만 착수하며, 하나라도 조건을 만족하지 못하면 만들지 않는다.

- 북쪽 기지 공통 도착 뒤 1~2초 확장 엔딩 연출
- `3개 항로 연결 완료` 완주 기록 강조
- 선택 엔딩 확장음 또는 크레딧 전환

선택 엔딩을 만들지 않아도 북쪽 기지 공통 도착과 항로 완료 상태는 반드시 동작한다.

## 4. P0 핵심에 포함하지 않는 것

- 전용 엔딩 화면·전용 엔딩 컷신·필수 크레딧 전환
- 캐릭터 표정 세트·캐릭터 애니메이션
- T4·T5, 흑빙·서리빙
- 자동 파쇄, 방치 수익, 앱 종료 중 재화 획득
- 정비 노드마다 다른 아이콘 13종, 노드별 선박 외형 교체
- 목적지마다 별도 플레이 배경·하역 애니메이션
- macOS, 온라인·클라우드 저장, Steam 기능, 분석 SDK
- 현금 결제, 시장·거래 경제

아래 순서로만 줄인다. 앞 항목을 자르지 않고 뒤 핵심을 먼저 제거하지 않는다.
