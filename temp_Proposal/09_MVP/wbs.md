예상시간은 AI 에이전트 실행만이 아니라 다음을 포함한다.

```text
Task 전달·프롬프트 작성
→ 에이전트 구현
→ 개발자 diff 검토
→ 수정 1회
→ Unity 컴파일·수동 검증
→ PR 작성·리뷰 대응
```

표의 Task 예상시간은 주담당자의 시간이다. 두 번째 이상의 큰 재작업과 선행 Task 대기는 포함하지 않는다. 다른 사람의 PR을 검토하는 시간은 일별 `LEAD` 합계에 별도 버퍼로 포함한다.

`ART`와 `PLAN`의 하루 가능 시간이 아직 정해지지 않았으므로 예상시간은 순수 작업시간 기준이다. 시작 전에 각자의 가능 시간을 확인하고, 시간이 부족하면 표의 우선순위가 높은 Task부터 수행한다.

## 11. P0 완료 정의

- 클릭·누르기·치명타로 T1~T3를 파괴할 수 있음
- 최대 20개 얼음, 결정빙·균열빙, 보조 파쇄·연쇄가 구현 잠금대로 동작
- 자금과 목적지 진행이 파괴 순간 한 번만 반영됨
- 정비 14개를 26개 구매 Step으로 탐색하고 구매·적용·저장할 수 있음
- 항해·쇄빙·정산·목적지 도착을 반복할 수 있음
- 북쪽 기지 도착 뒤 `운항 완료`와 쇄빙 시작 비활성
- 진행 중 종료를 복원하지 않으며 저장 완료된 결과를 유지함
- 800×72·960×540 창이 Windows에서 의도한 위치·포커스로 동작
- 표준·시연 데이터와 저장 파일이 분리됨
- 다른 Windows PC에서 처음부터 북쪽 기지까지 진행 가능
