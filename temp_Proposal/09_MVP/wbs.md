## 2. 난이도와 시간 기준

| 난이도 | 기준 |
|---:|---|
| 1 | 정적 데이터·문구·단순 배치. 실패 영향이 해당 View에 한정됨 |
| 2 | 독립 View 또는 낮은 위험의 단일 동작 |
| 3 | 한 모듈 안의 일반 기능. Unity 수동 테스트 필요 |
| 4 | 모듈 간 계약, 전투 예외, 저장 연동. 리드 리뷰 필수 |
| 5 | 전체 상태 흐름, 저장 복구, Windows 창, 최종 통합. 실기 검증 필수 |

예상시간은 AI 에이전트 실행만이 아니라 다음을 포함한다.

```text
Task 전달·프롬프트 작성
→ 에이전트 구현
→ 개발자 diff 검토
→ 수정 1회
→ Unity 컴파일·수동 검증
→ PR 작성·리뷰 대응
```

표의 Task 예상시간은 주담당자의 시간이다. 두 번째 이상의 큰 재작업과 선행 Task 대기는 포함하지 않는다. 다른 사람의 PR을 검토하는 시간은 일별 `LEAD` 합계에 별도 버퍼로 포함한다.

`ART`와 `PLAN`의 하루 가능 시간이 아직 정해지지 않았으므로 예상시간은 순수 작업시간 기준이다. 시작 전에 각자의 가능 시간을 확인하고, 시간이 부족하면 표의 우선순위가 높은 Task부터 수행한다.
