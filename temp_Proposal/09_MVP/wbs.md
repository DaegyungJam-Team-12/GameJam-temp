## 1. 역할과 소유권

| 담당 | 책임 | 직접 변경하는 것 | 변경하지 않는 것 |
|---|---|---|---|
| 민준 — 코어·진행·통합 | 상태 흐름·경제·정비·저장·창 전환·최종 통합 | 자금·목적지 진행·정비 구매·저장 상태 | 파괴 판정 |
| 시연 — 얼음 파쇄 게임플레이 | 입력·얼음·특수빙·보조 파쇄·연쇄 | 파괴 판정과 파괴 결과 전달 | 자금·목적지·구매·저장 상태 |
| 정환 — 표시·피드백·QA | HUD·보상 표시·정산 패널·버튼 상태·QA | 전달받은 값의 화면 표시 | 게임 수치·정산 합계 계산 |
| 아티스트 — 2D 비주얼 | 캐주얼 손그림풍 게임 에셋·선장 원화·P0 오디오 | 원본 PSD/KRA, Export PNG | Unity 씬·프리팹·임포트 설정 |

**게임 수치 변경은 민준, 파괴 판정은 시연, 전달받은 값의 화면 표시는 정환**이 맡아 같은 상태를 여러 곳에서 계산하지 않는다.

## 4. 통합 원칙

- 시연은 파괴 결과만 전달하고 자금·목적지 진행을 직접 변경하지 않는다.
- 민준만 자금·목적지 진행·정비 구매·저장 상태를 변경한다.
- 민준은 타이머 종료 시 읽기 전용 `SettlementSummary`를 고정한다.
- 정환은 전달받은 상태를 표시하지만 게임 수치와 정산 합계를 계산하지 않는다.
- 각 담당자는 가짜 데이터로 자기 기능을 먼저 확인할 수 있어야 한다.
- 실행 진입 연결과 최종 실행 파일은 민준이 담당한다. 별도 타이틀 화면 없이 실행 시 바로 축소 런처를 띄운다.
- 화요일 통합 뒤에는 새로운 핵심 시스템을 추가하지 않는다.
- 목·금에는 진행을 막는 오류, 가독성, 밸런스, 아트·오디오, 제출 안정성을 우선한다.

## 5. 완료 정의

- 7/19 일: T1 파괴 결과가 자금과 HUD까지 연결된다.
- 7/20 월: 항해부터 정산까지 한 사이클을 반복할 수 있다.
- 7/21 화: 정비 13개, 특수빙, 보조 파쇄, 연쇄가 1차 통합된다.
- 7/22 수: 저장 후 재실행과 처음부터 목적지 도달까지 가능한 Windows 빌드를 만든다.
- 7/23 목: 전체 플레이를 기준으로 밸런스·연출·아트·오디오를 보완한다.
- 7/24 금: 다른 PC 실행, 해상도, 저장, 정산 수치, 제출 파일을 최종 확인한다.
- 수요일 이후에는 새로운 핵심 시스템을 추가하지 않는다.
