> **상태: LOCKED · v1.0 · 2026-07-18**  
> 인원: 개발자 3명 + 2D 아티스트 1명

## 1. 역할과 소유권

| 담당 | 책임 | 직접 수정하는 영역 | 수정하지 않는 영역 |
|---|---|---|---|
| 개발자 1 — 파쇄 코어 | 얼음·입력·파괴·특수빙·성능 | 파쇄 스크립트, 독립 테스트 프리팹·테스트 | 메인 씬, UI 프리팹, 항로·세이브 규칙 |
| 개발자 2 — 진행·데이터 | 상태·정산·항로·정비·저장·밸런스 | 로직 스크립트, JSON, 자동 테스트 | 메인 씬, UI 프리팹, 얼음 VFX 구현 |
| 개발자 3 — 창·UI·통합 | 창 제어·UI·씬·아트/오디오·빌드 | 메인 씬, UI 프리팹, 임포트 설정, 빌드 | 파쇄·항로 규칙의 원천 로직 |
| 아티스트 — 2D 비주얼 | 캐주얼 핸드페인팅 게임 에셋·미형 선장 원화 | 원본 PSD/KRA, Export PNG, 아틀라스 원본 | Unity 씬·프리팹·임포트 설정 |

### 단일 소유 규칙

- 메인 씬, Canvas, UI 프리팹, Unity 아트·오디오 임포트는 개발자 3만 수정한다.
- `RunResult`, `RouteState`, `BalanceData`, `GameEvents`는 D1에 세 개발자가 합의하고 개발자 2가 관리한다.
- 담당 외 변경은 직접 수정하지 않고 소유자에게 요청한다.
- 아티스트는 720×480 합성 PNG로 먼저 검수받고, 개발자 3이 Unity에 반영한다.

## 7. 일일 통합 루틴

| 시간 | 작업 |
|---|---|
| 업무 시작 | 각자 전날 통합 빌드 확인, 오늘 완료 조건 1줄 공유 |
| 오후 중간 | 계약·에셋 변경이 있으면 소유자에게 전달, 직접 교차 수정 금지 |
| 하루 1회 | 개발자 1 → 개발자 2 → 개발자 3 순으로 통합 브랜치 병합 |
| 통합 직후 | 개발자 3이 메인 씬 연결, 네 명이 10분 핵심 플로우 확인 |
| 종료 전 | 차단 버그·내일 의존성·컷 후보만 기록 |

장시간 회의 대신 실행 가능한 최신 빌드를 기준으로 판단한다. 통합이 깨지면 신규 작업보다 복구를 우선한다.

## 8. 의존성 전달 규격

### 개발자 1 → 개발자 2·3

- 파괴 이벤트, `RunResult` 원천값, 특수 효과 테스트 프리팹
- VFX·스프라이트 교체 슬롯과 피벗/사이즈 가이드

### 개발자 2 → 개발자 1·3

- 공유 모델·이벤트, JSON 예제, 상태 전이표
- 정산·항로·구매·저장 API와 테스트 결과

### 아티스트 → 개발자 3

- 원본 PSD/KRA + 투명 PNG + 720×480 합성 미리보기
- 변경된 파일 목록, 피벗·9-slice·레이어 메모

### 개발자 3 → 팀

- 매일 통합 Windows 빌드와 캡처
- 씬 연결 오류, UI 넘침, 실제 크기 아트 수정 목록

## 9. 완료 정의

작업은 코드나 그림을 만들었을 때가 아니라 아래 조건을 만족할 때 완료다.

- 담당 영역에서 스스로 검증했고 경고·에러를 남기지 않음
- 공유 계약과 단일 출처 문서의 수치를 따름
- 개발자 3 통합 빌드에서 연결됨
- 다른 담당자의 소유 파일을 불필요하게 수정하지 않음
- 완료 조건을 재현할 테스트 또는 짧은 수동 체크를 남김

## 10. D14 제출물

- Windows 표준 빌드 (`standard`)
- 발표용 시연 빌드 (`demo`)
- 1페이지 조작·게임 흐름 안내
- 크레딧·폰트·오디오 소스 라이선스
- 발표용 게임 캡처 또는 30~60초 영상
- 원본 프로젝트와 아트 소스 백업
